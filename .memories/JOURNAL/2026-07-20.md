# 2026-07-20 — Стартовые наборы («касты»)

## Что сделано

Введена система стартовых наборов (starter kits, «касты») — разнообразие
стартового снаряжения вместо единого набора. Требование владельца, детали
согласованы в чате и записаны в `docs/07-decisions.md` §13.

### Дизайн (со слов владельца)
- Выдача — **выбор в лобби через GUI**; в меню всегда «Без касты» и «Случайная».
- Каст = **доп. предметы + своё стартовое золото** (перекрывает `start-gold`).
  Системные предметы (вилка/помощник/блок возрождения) — всегда.
- **Всё снаряжение почти сломано** (`damage` ~85–90%): двигатель игры — ремонт.
  Золото минимальное (4–12). Разница кастов — в РОЛИ, не в силе (бомжеватый §11).
- Глобальная библиотека-шаблон + **копии на арену** (правишь копию, не оригинал).

### Код
Новые файлы:
- `kit/Kit.java` — модель: name/icon/gold/lore/items. `load` читает и рукописный
  формат (`Items.fromSpec`), и сериализованные ItemStack (копия арены после GUI).
  `apply(Player)` — броня авто-надевается в слоты, остальное в инвентарь.
  `copy(newId)` — глубокая копия под копирование на арену.
- `kit/KitRegistry.java` — глобальный `kits.yml` (read-only, `saveResource` при
  отсутствии). Оригинал командами не пишется → рукописный формат и комментарии целы.
- `menu/KitSelectMenu.java` — выбор в лобби (сетка 54, касты + «Без касты» barrier
  + «Случайная» ender_pearl), выбранное подсвечивается `setEnchantmentGlintOverride`.
- `menu/KitEditorMenu.java` — админ-редактор предметов каста (как `LootEditorMenu`:
  разложил → пересобралось на закрытии).
- `resources/kits.yml` — 5 стандартных: warrior/archer/angler/miner/dealer.

Правки:
- `arena/Arena.java` — список кастов + `default-kit`, load/save `arenas/<id>/kits.yml`.
- `game/GameSession.java` — `chosenKit` (id|none|random), селектор в `join`,
  переписан `giveKit` (системка + золото/предметы каста через `resolveKit`),
  очистка в `cleanup`.
- `listener/GameListener.java` — клик по селектору `kit-select` в лобби.
- `command/EscapeCommand.java` — `/escape kit list|copy|create|delete|edit|name|
  icon|gold|default` + tab + help.
- `util/Keys.java` — `KIT_ID`. `EscapePlugin` — wiring KitRegistry (load/reload).
- `messages.yml` — секция `kit.*` + строки в help.admin.

## Грабли, на которые наступил

  LEARNINGS: `<gold><gold>` схлопывался). Переименовал в `goldval`. Урок закреплён.

# 2026-07-20 (позже) — Мусорщик (скупка сломанного)

## Что сделано
NPC, которому продают сломанное снаряжение за золото — замыкает петлю экономики
(ломается → чинить/сдавать → золото). Согласовано в чате, записано в §14.

### Дизайн (со слов владельца)
- **Цена растёт с износом**: `цена = max(1, round(база(материал) × доля_износа))
  × количество`. Чем ушатаннее — тем больше (металлолом). Годное платит копейки.
- **Принимает только изношенное**: прочность есть, износ ≥ `min-wear-percent`
  (дефолт 25), материал в прайсе.

### Код
- `trader/TraderType.java` — секция `scrap` (Map<Material,Integer> + порог),
  `isScavenger()`, load/save. Третья роль рядом с shop/overseer.
- `menu/ScavengerMenu.java` — click-to-sell: сканирует инвентарь, показывает
  сломанные вещи с ценой (PDC `SCRAP_SLOT` = слот инвентаря), клик = продать одну,
  «Сдать весь хлам» = всё. Предмет берётся из инвентаря → потерять нельзя.
  Guard `hasPluginData` отсекает системку (SPECIAL_ITEM/CONTRACT_ID/THEME_ID/
  RESPAWN_OWNER/MARKER_TYPE). Звук наковальни на продажу.
- `menu/NpcMenu.java` — переписан на динамические кнопки (до 3 ролей),
  центрирование 1/2/3 кнопок, маршрут на ScavengerMenu.
- `listener/GameListener.java` — открытие NPC: roles>=2 → NpcMenu, иначе прямой
  вход (в т.ч. scavenger-only без «назад»).
- `command/EscapeCommand.java` — addscrap (материал из руки + база), removescrap,
  scrapwear (%), scraplist + tab + help.
- `arena/ArenaCheck.java` — scavenger теперь валидная роль (не даёт WARNING).
- `util/Keys.java` — `SCRAP_SLOT`. `messages.yml` — секции `scavenger.*`,
  `npc.scavenger-button`, `admin.scrap-*` + help.admin.
- `docs/examples/traders-prison.yml` — пример типа `МУСОРЩИК`.

### Решения по объёму
- **Стата продаж не добавлялась**: `StatsRepository.COLUMNS` — whitelist, новая
  колонка = ALTER TABLE + миграция существующей stats.db + правки Row/SELECT/UI.
  Владелец не просил; отложено (только DebugLog фиксирует продажи).

## Проверка
- `./gradlew build` — BUILD SUCCESSFUL. Смоук не делался (сервера нет). Живьём не
  проверено (см. PLANS).
