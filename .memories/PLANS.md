  Полная замена системы «пер-ареновый пул + пер-ареновые категории-квоты» на
  **глобальные категории лута**. Спека утверждена владельцем поштучно (14 edge-
  кейсов + контракты + рефилл); ниже — то, что реализовано.

  **Модель.** Категория теперь ГЛОБАЛЬНАЯ (`loot/<id>.yml` в корне плагина) и
  несёт в себе И лут, И все лимиты. Пер-ареновый `loot.yml` и пер-ареновые
  `chest-categories.yml` УБИРАЮТСЯ (миграция — ниже). Нет категории у сундука =
  нет лута. Поля категории:
  - `name`, `icon`;
  - `weight` 1..1000 — вес среди категорий;
  - `min/max-per-chest` — слотов в одном сундуке (−1 = без огр.);
  - `min/max-per-arena` — слотов этой категории на всю арену (−1 = без огр.);
  - `min/max-chests` — сколько сундуков этой категории за матч (−1 = без огр.);
  - `refill-seconds` — свой рефилл у каждой категории;
  - `loot` — список `{item, weight, amount, wear}` (вес внутри категории).
  Везде `−1` = без ограничения.

  **Привязка точек — многокатегорийная.** `Arena.chestSpots` становится
  `Map<Location, List<String>>` (0..N категорий на точку). Формат `locations.yml`:
  `{location, categories:[...]}`. Обратная совместимость чтения: одиночный
  `category`/плоский Location → список из одного/пустой. Мастер-жезл: ПКМ жезлом
  ТОГГЛИТ категорию на текущей точке (можно навесить несколько), отдельное
  действие «дальше».

  **Спавн (3 фазы) в GameSession.placeChests:**
  1. *Пулы:* точка → её набор категорий. Пустой набор → воздух, сундука нет.
  2. *Раздача точек по категориям (≤1 на точку); конечный `max` — первым,
     филлеры `max=-1` — последними:*
     - Проход 1 (минимумы, по убыванию веса): категория берёт до `min-chests`
       из свободных кандидатов (рандом). Свободных < min → берёт сколько есть,
       лог (кейс 4). Больший вес резервирует раньше (кейс 5).
     - Проход 2 (добор, взвешенный рандом по весу категории): свободная точка →
       одна из её конечных категорий с местом до `max-chests` (кейс 2/3).
     - Проход 3 (филлеры `max=-1`): все оставшиеся свободные точки → их
       безлимитным категориям (кейс 6).
     - Оставшиеся свободные точки → воздух.
     Выбранные точки = `CHEST` (rememberEditedBlock!), остальные воздух → restore
     после матча.
  3. *Заполнение:* слотов = ролл `[min,max]-per-chest` (−1→0/27, верх режется 27,
     кейс 9); взвешенно-случайные предметы из лута категории. Бюджет
     `min/max-per-arena` (слоты, на всю арену): `max` МЯГКИЙ — сундук всё равно
     кладёт свой `min-per-chest` сверх него (кейс 10); арена-min недостижим →
     лог, без досоздания сундуков (кейс 8). Категория с 0 предметами → сундук
     ПУСТОЙ (кейс 11).

  **Рефилл.** Игнорирует арена-бюджет полностью. НЕ чистит сундук: докидывает в
  свободные слоты свежий ролл `[min,max]-per-chest` из лута категории, старое
  лежит. Таймер взводится, когда игрок ЧТО-ТО забрал (не когда пусто дочиста),
  перевзвод на следующий забор. `refill-seconds` — из категории.

  **Контракты.** Конфиг в `arena.yml` → `contracts: {min/max-per-arena,
  min/max-per-chest}`. Кладутся НЕЗАВИСИМО от лута, в бюджет категорий НЕ идут.
  Раздача: общее число = ролл `[min,max]-per-arena` (зажат `активные×max-per-chest`);
  кладём ПОСЛЕ лута в свободные слоты (сначала каждому `min-per-chest`, потом
  добор до `max-per-chest`); нет места на минимум — лог. Пустые сундуки —
  кандидаты под контракты.

  **Динамические сундуки** — оставляем (fallback для больших арен): немаркированный
  сундук при открытии наследует категорию с НАИБОЛЬШИМ весом среди глобальных
  категорий, у которых есть лут; вне бюджета.

  **Миграция (автоматическая).** Из старого `arenas/<id>/loot.yml` +
  `chest-categories.yml` создаём глобальные `loot/<id>.yml`-сиды (лут = старый
  пул арены, лимиты из старых `loot-min/max`→per-chest, `refill-seconds`,
  `quota`→ориентир `max-chests`). Точки арены (`chestSpots`) ремапятся на новые
  глобальные id (префикс арены, чтобы не коллизить). Старые пер-ареновые файлы
  после конвертации не используются.

  **GUI** `/escape loot` (замена `chestcat`): экран «Категории» (paginated:
  иконка+лор с весом/лимитами/refill; клик=редактор; «Создать» из инвентаря / из
  просматриваемого сундука / копией; удалить/переименовать) → «Редактор
  категории» (верхний ряд = настройки кликом ±/чат; сетка = предметы с
  весом/кол-вом/износом, ЛКМ правка, Shift удалить, перетаскивание добавить).
  База — `menu/PaginatedEditorMenu`.

  **Новые/меняемые файлы:** новые `loot/LootCategory`, `loot/LootCategoryRegistry`,
  `loot/LootMigration`; меняются `arena/Arena` (chestSpots→multi, contracts,
  выпил loot/per-arena-cat), `game/GameSession` (спавн/рефилл/контракты/dynamic),
  `arena/ChestSetupManager`+`arena/WizardState`+`menu/ChestSetupMenu` (мультитоггл),
  `menu/LootEditorMenu` (редизайн), `command/EscapeCommand` (`loot`-команды),
  `EscapePlugin` (реестр), `Keys` (при нужде), `messages.yml`. Старые
  `category/ChestCategory`+`category/CategoryRegistry`+`resources/chest-categories.yml`
  — удаляются/замещаются.
