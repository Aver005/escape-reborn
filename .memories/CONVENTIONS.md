# CONVENTIONS — стиль и инварианты

Last updated: 2026-07-18

## Стиль кода

- Java 25, Paper API; **никакого NMS/рефлексии** — только публичный API.
- Скобки Allman (открывающая на своей строке) — как во всём написанном коде;
  короткие guard-ветки в одну строку: `if (x == null) {return;}`.
- Javadoc/комментарии — по-русски, только там, где код сам не объясняет.
- Пакеты по подсистемам (`arena/`, `game/`, `contract/`...), без god-utils.
- **Никаких инлайновых FQN в коде** (`net.kyori.adventure.bossbar.BossBar x`) —
  всегда import. Группы импортов: java → me.aver005 → net.kyori → org.bukkit.
  При коллизии имён (например, adventure `BossBar` vs `org.bukkit.boss.BossBar`,
  наш `menu/Menu` vs чужие) — импортируй используемый в файле чаще, второй
  пиши полностью; в одном файле оба одноимённых класса стараться не смешивать.

## Инварианты (не ломать)

1. **Тексты игрокам — ТОЛЬКО из messages.yml** (MiniMessage). Ни одной
   захардкоженной строки для игрока в Java. Новый текст = новый ключ.
2. **Логи сервера — только ASCII** (Windows-консоль портит кириллицу).
3. **Данные на предметах/сущностях — только PDC** (`util/Keys`). Парсить
   лор/имена запрещено (лор — отображение). Новый ключ регистрируй в `Keys`.
4. **Любое изменение блока мира в матче** — сначала
   `session.rememberEditedBlock(block)` (или putIfAbsent в editedBlocks),
   иначе cleanup не откатит.
5. **Заспавненные сущности и дропы** — регистрируй UUID в сессии
   (`spawnedEntities` / `trackDrop`), иначе останутся после матча.
6. **GUI — только через `menu/Menu`** (InventoryHolder). Не сравнивай заголовки.
7. **Игровая логика — main thread.** Async — только SQLite и AsyncChatEvent
   (уже реализовано); не добавляй async-обращений к Bukkit API.
8. **Снапшот игрока — неприкосновенен**: вход в сессию = `PlayerSnapshot.save` → `clear`;
   любой выход из сессии (leave/смерть матча/cleanup/onJoin-recovery) должен
   заканчиваться `restore`. Игрок никогда не должен «унести» игровые предметы.
9. **Конфиги арен пишутся через `ArenaManager.save(arena)`** после каждого
   админ-изменения — не копи изменения в памяти.
10. **Статистика — только через `StatsRepository`** (колонки из whitelist
    `COLUMNS`); прямых SQL-строк вне класса нет.

## Процесс

- Проверка изменений: `./gradlew deploy` → перезапуск `E:\Servers\escape\start.bat`
  → лог без стектрейсов + ручная проверка фичи в игре.
- После нетривиального изменения — обнови `.memories/STATE.md` + JOURNAL
  (см. INDEX.md, «Правило поддержки»).
- Поведение игры должно соответствовать `docs/07-decisions.md`; расхождение —
  либо баг кода, либо осознанное решение, которое надо туда дописать.
