## [DONE] Зависание сервера: >28 товаров у типа NPC — исправлено 2026-07-19

Найдено чтением кода при написании `docs/guides/`. `ShopMenu` (конструктор) при
открытии магазина уходил в **бесконечный цикл на main thread**, если у типа NPC
больше 28 трейдов:

```java
while (isBorder(slot)) {slot++;}          // slot >= size-9 => isBorder всегда true
if (slot >= inventory.getSize()) {break;} // недостижимо: while не выходит
```

`isBorder` возвращает `true` для любого `slot >= size - 9`, поэтому `while`
крутился вечно, а проверка выхода стояла **после** него.

**Фикс** → `menu/ShopMenu.java`: граница внесена в условие цикла
(`while (slot < inventory.getSize() && isBorder(slot))`). Ёмкость окна вынесена
в `ShopMenu.MAX_TRADES = 28` (6 рядов, внутренние 4x7), лимит темок — в
`ThemesMenu.MAX_THEMES = 7` (слоты 10-16). `ArenaCheck` получил два WARNING —
`npc-trades-overflow` и `npc-themes-overflow`, чтобы лишний контент не пропадал
молча.

Проверено: симуляция раскладки слотов для 0..200 товаров (старый цикл виснет
ровно на 29, новый всегда завершается и кладёт `min(n, 28)`; для n <= 28
поведение идентично — регрессии нет) + смоук на `E:\Servers\escape`
(`Escape enabled`, `Done (12.0s)`, 0 стектрейсов).

**Остаётся by design:** больше 28 товаров и больше 7 темок физически некуда
показать — лишние не отображаются. Пагинации нет.

## Ограничения by design

- **Максимум 28 товаров (`ShopMenu.MAX_TRADES`) и 7 темок
  (`ThemesMenu.MAX_THEMES`) на тип NPC**: лишнее физически некуда показать,
  пагинации нет. Сданные темки продолжают занимать слоты. С 2026-07-19
  `/escape check` предупреждает о превышении (только для NPC, реально
  расставленных на арене).
- `[?]` **Двойные сундуки не поддержаны**: LOOT-прогресс, FIND-пересчёт и
  рефилл ловят только `holder instanceof Chest` → точки сундуков не ставить
  вплотную друг к другу. → `listener/GameListener.onInventoryOpen/Close`.
- **Защита от огня/листвы действует на весь мир арены** (как в оригинале) —
  аренам нужны выделенные миры, не основной выживач. → `ProtectionListener`.
- **Спавнов должно быть ≥ игроков**: при нехватке точки переиспользуются
  (шафл по кругу), игроки могут заспавниться вместе. → `GameSession.startMatch`.
- **Кредит убийства** — окно 10 сек после последнего удара (`KILL_CREDIT_MILLIS`);
  смерть от падения без чужого урона убийцей не считается.
- **Restart сервера посреди матча**: мир НЕ откатится автоматически (editedBlocks
  в памяти). Игроки восстановятся при заходе (снапшоты на диске), но сломанные
  решётки/выставленные сундуки останутся до ручной чистки или следующего матча
  на тех же точках. `[IDEA]` — журналить editedBlocks на диск.
