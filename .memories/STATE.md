- **лут**: рукописный формат loot.yml ({type,weight,amount}), случайный износ
  wearmin/wearmax % (дефолт 40–90), пример пула docs/examples/loot-prison.yml;
- **динамические сундуки** (`/escape set ID dynamicchests 1`, дефолт выкл):
  немаркированный сундук при открытии в матче получает лут и становится
  игровым (рефилл/LOOT/ключик/табличка); исходное содержимое возвращается
  после матча; двойные сундуки не поддержаны (как и у обычных);
- **категории сундуков + мастер настройки** (§16, собрано 2026-07-20, смоук НЕ
  делался): у арены есть категории (`chest-categories.yml`, копии глоб. шаблонов) →
  отбор по квотам (из точек категории случайно `min(round(quota*mult), точек)`);
  `loot-min/max` задаёт число слотов, `refill-seconds`=0 = без рефилла;
  `chest-count-mult` крутит квоты. Нет категорий = прежний режим (chest-count).
  Мастер `/escape chestsetup` (стеш инвентаря PlayerSnapshot, ADVENTURE, жезлы,
  телепорт по точкам, ПКМ = категория+переход, барьер/stop = выход, GUI-прыжок);
  крашоустойчивость через onJoin-восстановление; вход на арену под мастером
  заблокирован. Команды `/escape chestcat ...`, ArenaCheck-ветка для квот;
- **NPC-конфиги**: рукописный формат traders.yml ({type,amount,price} + themes),
  примеры docs/examples/traders-prison.yml + themes-prison.yml;
- **блоки-подсказки точек** (`arena/SetupMarkers`): спавн — 2 серых стекла,
  финальный спавн — 2 чёрных, точка NPC — 2 розовых, точка сундука — настоящий
  сундук (лут в нём не учитывается и чистится на старте/после матча). Слом
  любого блока подсказки вне матча = удаление точки (`escape.admin`, арена не в
  игре); подсказки снимаются в `startMatch` ДО телепорта и возвращаются в
  `cleanup` после отката `editedBlocks`; `/escape markers <ID>` расставляет их
  заново (в воздух/заменяемый блок, карту не затирает);
- **улучшенные ванильные механики** (`listener/MechanicsListener`, только в
  сессии): снежки бьют на 2.0, яйца на 3.0 (`config.yml` → `projectiles`,
  0 = ваниль); урон выдаётся из `ProjectileHitEvent` источником THROWN, поэтому
  работает броня, «Кровавая луна», кредит убийце и фейковая смерть, а ванильное
  окно неуязвимости не даёт спамить в упор. Куры из яиц в мирах арен
  подавляются (`ProtectionListener`). **Удочка-крюк**: `PlayerFishEvent`
  CAUGHT_ENTITY тянет игрока к рыбаку, сила = base + per-level × уровень
  «Силы» (POWER) на удочке (`config.yml` → `fishing-hook`); скорость ставится
  СЛЕДУЮЩИМ тиком, чтобы перебить ванильный `pullEntity`; падение после крюка
  засчитывается рыбаку. Зачарованная удочка — только от админа
  (`enchants: {power: N}` в loot/traders или `/give`);
- **NPC 2.0**: смотрящие с темками (8 типов, сдача SELF/ANY/адресату,
