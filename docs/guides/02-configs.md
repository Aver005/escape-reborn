# Справочник конфигов

Все файлы плагина, все ключи, дефолты и правила безопасного редактирования.

---

## Где что лежит

```
plugins/Escape/
├── config.yml            — глобальные настройки и дефолты новых арен
├── messages.yml          — ВСЕ тексты игрокам (MiniMessage)
├── contracts.yml         — контракты (глобально)
├── themes.yml            — темки (глобально)
├── traders.yml           — типы NPC: торговцы и смотрящие (глобально)
├── stats.db              — статистика (SQLite)
├── arenas/
│   └── <ID>/
│       ├── arena.yml     — настройки арены
│       ├── locations.yml — точки (спавны, сундуки, руды, рычаги, NPC, столы)
│       └── loot.yml      — пул лута арены
└── snapshots/
    └── <uuid>.yml        — снапшот игрока на время матча (страховка от краша)
```

Дефолтные `config.yml` и `messages.yml` распаковываются из jar при первом
запуске. Остальные файлы создаются по мере работы админ-команд.

---

## Золотое правило редактирования

> **Плагин держит все конфиги в памяти и перезаписывает их на диск при
> выключении сервера и по `/escape save`.** Если отредактировать файл при
> работающем сервере и не сделать `/escape reload`, правки будут затёрты.

### Безопасные сценарии

| Сценарий | Порядок действий |
|---|---|
| **Лучший** — сервер выключен | Остановить сервер → править файлы → запустить |
| Сервер работает, матчей нет | Править файлы → **сразу** `/escape reload` |
| Сервер работает, идёт матч | ⛔ Не редактировать. `/escape reload` **останавливает все матчи** |

### Что затирает правки

| Действие | Что перезапишет |
|---|---|
| Выключение сервера (`onDisable`) | Все арены, contracts.yml, themes.yml, traders.yml |
| `/escape save` | То же |
| Любая изменяющая команда по арене | `arena.yml` + `locations.yml` + `loot.yml` **этой** арены |
| Любая команда `createcontract`/`contract*`/`addcontract` | `contracts.yml` |
| Любая команда `createtheme`/`theme*` | `themes.yml` |
| Любая команда `createvillager`/`villagername`/`addtrade`/`addtheme`/`removetheme` | `traders.yml` |
| Закрытие GUI-редактора лута или трейдов | `loot.yml` / `traders.yml` |

`messages.yml` плагин **никогда не перезаписывает** — его можно править
свободно, достаточно `/escape reload` для применения.

### Потеря комментариев и рукописного формата

Bukkit-YAML при сохранении **выбрасывает комментарии** и разворачивает
компактную запись. Кроме того, рукописный формат предметов читается, но
сохраняется в сериализованном виде:

| Файл | Пишете вы | Станет после первого сохранения |
|---|---|---|
| `loot.yml` | `- {type: STONE_SWORD, weight: 20}` | `- {item: {==: org.bukkit.inventory.ItemStack, ...}, weight: 20}` |
| `traders.yml` | `- {type: BREAD, amount: 3, price: 2}` | `- {item: {==: org.bukkit.inventory.ItemStack, ...}, price: 2}` |

Функционально это одно и то же, но читаемость теряется навсегда.

**Вывод:** держите рукописные исходники пулов лута и ассортиментов **вне**
`plugins/Escape/` (например, в репозитории карты) и копируйте их на место при
выключенном сервере. Готовые примеры — в [`docs/examples/`](../examples/).

## `config.yml` — глобальные настройки

### `defaults` — дефолты **новых** арен

Применяются **только в момент `/escape create`**. Изменение этих значений
задним числом **не влияет** на уже созданные арены — их правьте через
`/escape set`.

```yaml
defaults:
  min-players: 2
  max-players: 12
  chest-count: 75
  trader-count: 32
  table-count: 5
  duration-seconds: 1200        # 20 минут
  event-interval-seconds: 210
  salary-interval-seconds: 600
  salary-gold: 16
  glow-seconds-before-end: 600
  glow-bonus-gold: 18
  start-delay-seconds: 60       # отсчёт при наборе минимума
  start-delay-full-seconds: 10  # отсчёт при полной арене
  fork-uses: 4                  # прочность вилки в решётках
  start-gold: 24
```

> ⚠️ **`wear-min-percent`, `wear-max-percent` и `dynamic-chests` в секции
> `defaults` не читаются.** Добавлять их туда бесполезно — новые арены всегда
> получают жёстко зашитые `40` / `90` / `false`. Задавайте через
> `/escape set <ID> wearmin|wearmax|dynamicchests`.

### `assistant` — кулдауны «Личного проводника», секунды

```yaml
assistant:
  nearest-chest: 150   # ближайший сундук
  random-player: 240   # случайный игрок
  nearest-trader: 180  # ближайший торговец
  places: 90           # меню отмеченных локаций
```

Если ключ отсутствует, используется значение **120**. Кулдауны считаются
отдельно для каждого игрока и каждой способности, живут в рамках одного матча.

Способность «случайный игрок» дополнительно требует **минимум 3 живых игрока** —
иначе выдаёт отказ и кулдаун не тратится.

### Прочие глобальные ключи

```yaml
chest-refill-seconds: 180        # период рефилла сундуков

respawn-block:
  lightning-interval-seconds: 300  # молния по всем установленным блокам
  respawn-delay-seconds: 5         # пауза в спектаторах перед возрождением

match:
  start-xp-levels: 50   # стартовые уровни XP (валюта зачарования)
  kill-xp-levels: 10    # +уровней за убийство

themes:
  drop-cooldown-seconds: 60  # кулдаун на новую темку после броска
  key-chance: 0.10           # шанс «Волшебного ключика» за сдачу (0..1)

offline-guard:
  zombie-seconds: 90    # сколько стоит AFK-страж
  return-seconds: 240   # общее окно возврата от момента выхода
```

Эти параметры **глобальные** — задать их отдельно для конкретной арены нельзя.

## `arenas/<ID>/arena.yml`

Пишется автоматически после каждой админ-команды по арене. Все числовые ключи
меняются через `/escape set` — см. [01-commands.md](01-commands.md#админ-числовые-настройки-арены).

```yaml
display-name: '<green>Тюрьма Аркхем'   # MiniMessage, /escape setname
description: Классическая карта        # MiniMessage, /escape setdesc
world: prison                          # имя мира, /escape create | setlobby
enabled: true                          # /escape enable | disable
min-players: 2
max-players: 12
chest-count: 75
trader-count: 32
table-count: 5
duration-seconds: 1200
event-interval-seconds: 210
salary-interval-seconds: 600
salary-gold: 16
glow-seconds-before-end: 600
glow-bonus-gold: 18
start-delay-seconds: 60
start-delay-full-seconds: 10
fork-uses: 4
start-gold: 24
wear-min-percent: 40
wear-max-percent: 90
dynamic-chests: false
lobby:                                  # сериализованная Location
  ==: org.bukkit.Location
  world: prison
  x: 100.5
  y: 64.0
  z: 100.5
  yaw: 0.0
  pitch: 0.0
contracts:                              # /escape addcontract
  - КОПАТЕЛЬ
  - ОХОТНИК
dead-messages: []                       # шуточные сообщения о смерти
```

### Ключи, недоступные из команд

| Ключ | Как задать | Смысл |
|---|---|---|
| `dead-messages` | **только вручную** | Список шуточных фраз о смерти игрока. Пустой список → берутся дефолты из `messages.yml` → `death-messages` |
| `contracts` | добавить — `/escape addcontract`, **убрать — только вручную** | Список ID контрактов арены |

### Подводные камни

- **`fork-uses` при ручном создании `arena.yml`:** если ключ отсутствует,
  подставляется **1**, а не `4` из `config.yml`. Дефолты config.yml работают
  только через `/escape create`.
- **`world`** должен указывать на загруженный мир. Мир грузится раньше плагина —
  если мир подключается плагином-мультимиром с задержкой, арена не найдёт его
  и `check` выдаст CRITICAL.
- **`lobby`** сериализуется вместе с миром. Ручная правка координат возможна, но
  проще выполнить `/escape setlobby`.

## `arenas/<ID>/locations.yml`

Единственный способ **удалить** ошибочную точку — команд удаления нет.

```yaml
spawns:                    # /escape addspawn — точки старта
  - ==: org.bukkit.Location
    world: prison
    x: 10.0
    y: 64.0
    z: 10.0
    yaw: 0.0
    pitch: 0.0
final-spawns: []           # /escape addfinalspawn — точки финальной битвы
chests: []                 # /escape addchest
tables: []                 # /escape addtable
ores: []                   # /escape addore
levers:                    # /escape addlever — точка + имя локации
  - location:
      ==: org.bukkit.Location
      world: prison
      x: 20.0
      y: 64.0
      z: 20.0
    name: Столовая
traders:                   # /escape addvillager — точка + ID типа NPC
  - location:
      ==: org.bukkit.Location
      world: prison
      x: 30.0
      y: 64.0
      z: 30.0
    type: БАРЫГА
```

**Условия и детали:**

- `final-spawns` пустой — финальная битва использует обычные `spawns`.
- Имя рычага (`name`) должно **посимвольно совпадать** с `idle` контракта или
  темки типа `ACTIVATE`. Валидатор сверяет именно строки.
- `type` у трейдер-точки должен совпадать с ключом в `traders.yml` **с учётом
  регистра**.
- Все `Location` требуют загруженного мира на момент чтения. Точки в
  незагруженном мире молча пропадают из списка при загрузке.

## `arenas/<ID>/loot.yml`

Пул предметов, из которого наполняются сундуки.

### Рукописный формат (рекомендуется)

```yaml
items:
  - {type: WOODEN_PICKAXE, weight: 40}
  - {type: STONE_SWORD, weight: 18}
  - {type: ARROW, weight: 10, amount: 3}
```

| Поле | Обязательно | Смысл |
|---|---|---|
| `type` | да | Имя `Material`. Нераспознанные и `AIR` **молча пропускаются** |
| `weight` | нет (по умолчанию 1) | Вес в рулетке, чем больше — тем чаще. Минимум 1 |
| `amount` | нет (по умолчанию 1) | Размер стака. Минимум 1 |

### Сериализованный формат

Появляется автоматически после любого сохранения арены и после
`additem`/`edititems`. Поддерживает зачарования, кастомные имена, NBT:

```yaml
items:
  - item:
      ==: org.bukkit.inventory.ItemStack
      v: 4440
      type: STONE_SWORD
    weight: 18
```

Оба формата можно смешивать в одном файле.

### Износ

Прочность руками задавать не нужно — арена сама раскатывает каждому выданному
предмету случайный износ в диапазоне `wear-min-percent`–`wear-max-percent`
(дефолт 40–90 %, `0` выключает). Предметы, добавленные через `additem` с уже
выставленным уроном, не трогаются.

Готовый «тюремный» пул на 40+ позиций — [`docs/examples/loot-prison.yml`](../examples/loot-prison.yml).

## `traders.yml` — типы NPC

Глобальный файл: один тип можно ставить на любое число арен.

```yaml
traders:
  БАРЫГА:
    name: '<gold>Барыга'          # MiniMessage, /escape villagername
    trades:                        # ассортимент магазина
      - {type: BREAD, amount: 3, price: 2}
      - {type: LAPIS_LAZULI, amount: 4, price: 6}
    themes:                        # ID темок, /escape addtheme
      - МАЛЯВА
```

| Поле | Смысл |
|---|---|
| `name` | Отображаемое имя NPC (MiniMessage). По умолчанию — ID |
| `trades` | Список товаров. Есть хоть один → NPC является **магазином** |
| `themes` | Список ID темок. Есть хоть одна → NPC является **смотрящим** |

**Формат товара** (рукописный): `type` — материал, `amount` — количество
(по умолчанию 1), `price` — цена в золоте (минимум 1, дробные округляются вниз).
Сериализованный вариант — `{item: <ItemStack>, price: N}`.

**Роли NPC:**

| Есть `trades` | Есть `themes` | Роль |
|---|---|---|
| ✅ | ❌ | Торговец — ПКМ открывает магазин |
| ❌ | ✅ | Смотрящий — ПКМ открывает список темок |
| ✅ | ✅ | Совмещённый — ПКМ открывает выбор «Магазин / Темки» |
| ❌ | ❌ | Бесполезен, `check` даёт WARNING |

Пример на 6 тюремных NPC — [`docs/examples/traders-prison.yml`](../examples/traders-prison.yml).

## `themes.yml` — темки

```yaml
themes:
  МАЛЯВА:
    type: COURIER          # /escape themetype
    idle: ''               # цель: материал или имя рычага
    description: '<gray>Отнеси передачку Кладовщику'
    amount: 1              # /escape themeamount
    gold: 25               # /escape themegold, допустим 0
    turn-in: КЛАДОВЩИК     # /escape themereturn: SELF | ANY | <ID типа NPC>
```

| Поле | Дефолт | Смысл |
|---|---|---|
| `type` | — | Тип задания. Без него темка не выдаётся |
| `idle` | `''` | Цель: имя `Material` (MINE/BREAK/FIND/DELIVERY) или имя рычага (ACTIVATE) |
| `description` | `''` | Текст в меню (MiniMessage) |
| `amount` | 1 | Сколько раз выполнить. `≤ 0` → темка не выдаётся |
| `gold` | 0 | Награда золотом |
| `turn-in` | `SELF` | `SELF` — сдать выдавшему, `ANY` — любому смотрящему, `<VID>` — конкретному типу NPC |

**Условие выдачи темки** (`Theme.isComplete`): задан `type`, `amount > 0`, и —
для `COURIER` — `turn-in` указывает на **конкретный тип NPC** (не `SELF` и не
`ANY`). Передачку некому нести, если адресат не задан, поэтому такая темка молча
не выдаётся.

Разбор всех 8 типов — [04-arena-content.md](04-arena-content.md#темки).
Пример — [`docs/examples/themes-prison.yml`](../examples/themes-prison.yml).

## `contracts.yml` — контракты

```yaml
contracts:
  КОПАТЕЛЬ:
    type: MINE                              # KILLS|ACTIVATE|MINE|FIND|BREAK|LOOT
    idle: IRON_ORE                          # материал или имя рычага
    description: '<gray>Добудьте 3 железные руды'
    amount: 3
    price: 20                               # награда золотом
```

Контракт попадает в матч, только если его ID перечислен в `contracts:` файла
`arena.yml` соответствующей арены.

## ⚠️ Регистр ID в глобальных реестрах

**ID арен** нечувствительны к регистру — плагин приводит их к верхнему.

**ID контрактов, темок и типов NPC — чувствительны к регистру.** Реестры ищут по
точному совпадению ключа, а команды переводят введённый аргумент в ВЕРХНИЙ
регистр. Следствие:

```yaml
contracts:
  копатель:      # ← записано строчными вручную
    type: MINE
```

```
/escape contracttype копатель MINE   → ищет "КОПАТЕЛЬ" → «контракт не существует»
```

Такой контракт загрузится и будет работать в игре, но станет **недоступен для
всех команд**. Всегда пишите ID глобальных сущностей ЗАГЛАВНЫМИ.

## `messages.yml` — тексты

Все без исключения тексты игрокам. Формат — [MiniMessage](https://docs.advntr.dev/minimessage/format.html).
Плагин этот файл **никогда не перезаписывает**: правьте свободно, применяйте
`/escape reload`.

> **Файл сливается с версией из jar.** Ключ, которого нет в вашем файле,
> подтягивается из встроенных дефолтов. Следствия: после обновления плагина
> новые тексты работают **без ручного слияния**, и вы можете держать в файле
> только те строки, которые действительно изменили.

### Секции

| Секция | Содержимое |
|---|---|
| `prefix` | Префиксы: оповещение, смерть, победа, событие |
| `menu` | GUI выбора арены |
| `lobby` | Лобби, отсчёт, разминка |
| `game` | Матч: старт, зарплата, боссбар-таймер, свечение, финал, победа |
| `events` | Все случайные события |
| `assistant` | «Личный проводник» и его способности |
| `contract` | Бумаги-контракты |
| `chest` | Сундуки и рефилл |
| `shop` | Магазин NPC |
| `death-messages` | Шуточные сообщения о смерти (fallback для `arena.yml` → `dead-messages`) |
| `chat` | Каналы чата |
| `stats` | Вывод `/escape stats` |
| `info` | Текст `/escape info` (список строк) |
| `errors` | Все ошибки команд |
| `admin` | Ответы админ-команд, подсказки цепочек настройки |
| `respawn-block` | Блоки возрождения |
| `offline-guard` | Оффлайн-стражи |
| `check` | Отчёт валидатора |
| `debug` | Ответы `/escape debug` |
| `npc`, `theme` | NPC 2.0 и темки |
| `marker-types` | Названия типов маркеров |
| `trade-editor`, `loot-editor` | GUI-редакторы |
| `help` | Справка (`help.player`, `help.admin` — списки строк) |

### Правила редактирования

- **Плейсхолдеры** вида `<player>`, `<n>`, `<arena>`, `<seconds>` подставляются
  плагином. Удалить лишний можно, **придумать новый — нельзя**: неизвестный тег
  останется в тексте как есть.
- **Имя плейсхолдера не должно совпадать с тегом MiniMessage.** Именно поэтому
  числа передаются как `<n>`, а не `<gold>` — иначе тег цвета затеняется по всей
  строке и `<gold>золота` превращается в число.
- Ключи-**списки** (`help.player`, `info`, `death-messages`, `*-lore`) должны
  оставаться списками — превращать их в строку нельзя.
- **Опечатка в имени ключа ведёт себя по-разному.** Обычная строка выводится
  игроку красной заглушкой `[нет строки: имя.ключа]` — заметно сразу. А список
  (лор предмета, `info`, `help`) просто **исчезает без предупреждения**.
- Цвета — только MiniMessage-теги (`<green>`, `<gradient:...>`), legacy-коды
  `&a` не поддерживаются.
- Одиночный символ `<` в тексте трактуется как начало тега — экранируйте `\<`.
- Курсив у имён и лора снимается автоматически, писать `<!italic>` не нужно.
- **Ники и реплики игроков вставляются как литералы** — игрок не может
  протащить цвет или `<hover>` через свой ник. А вот `display-name` и
  `description` арены, имена NPC и описания заданий разбираются как
  MiniMessage: это ваш текст, ему доверяют.

### Имена и лор предметов менять безопасно

Плагин опознаёт свои предметы по скрытым меткам, а **не по названию или лору**.
Имя «Вилки», блока возрождения, «Волшебного ключика» и любого другого игрового
предмета можно менять как угодно — вплоть до другого языка и другого числа строк
лора — логика не сломается.

## Служебные данные

### `snapshots/<uuid>.yml`

Полный снапшот игрока (инвентарь 41 слот, здоровье, голод, эффекты, режим,
позиция), снятый при входе в лобби. Удаляется при возврате игрока.

**Не трогать руками.** Файл на диске — страховка от краша: если сервер упал во
время матча, при следующем заходе игрок восстанавливается из этого файла.
Удаление файла = потеря вещей игрока.

Файлы, «зависшие» после краша, восстановятся сами при заходе владельца.

### `stats.db`

SQLite со статистикой: победы, поражения, убийства, смерти, сыгранные матчи,
MVP, квесты, трейды, добытые руды, лучший и последний результат по убийствам.
Пишется асинхронно. Читается командой `/escape stats`.

Сброс статистики — удалить файл при выключенном сервере.

---

## Кто и когда пишет файлы

| Файл | Пишется | Читается |
|---|---|---|
| `config.yml` | никогда (только распаковка при первом старте) | `onEnable`, `/escape reload` |
| `messages.yml` | никогда | `onEnable`, `/escape reload` |
| `arenas/<ID>/*` | после каждой админ-команды по арене, `/escape save`, `onDisable` | `onEnable`, `/escape reload` |
| `contracts.yml` | после команд контрактов, `/escape save`, `onDisable` | `onEnable`, `/escape reload` |
| `themes.yml` | после команд темок, `/escape save`, `onDisable` | `onEnable`, `/escape reload` |
| `traders.yml` | после команд NPC, `/escape save`, `onDisable` | `onEnable`, `/escape reload` |
| `snapshots/<uuid>.yml` | вход игрока в лобби | выход игрока / вход после краша |
| `stats.db` | по ходу матча, асинхронно | `/escape stats` |
