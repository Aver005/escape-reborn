# Создание арены: пошагово с ветвлением

От пустого мира до арены, на которую пускают игроков. Справочники команд и
конфигов — [01-commands.md](01-commands.md), [02-configs.md](02-configs.md).

---

## Прежде чем начать

| Требование | Почему |
|---|---|
| **Отдельный мир под арену** | Защита от огня, распространения блоков и гниения листвы действует на **весь мир** арены. Основной выживач в тот же мир класть нельзя |
| Право `escape.admin` | Все команды настройки |
| Вы в игре, не в консоли | Команда `/escape` работает **только от игрока** |
| Карта уже построена | Точки ставятся по готовой геометрии |

**Что откатывается после матча, а что нет.** Плагин запоминает и возвращает
только те блоки, которые изменил сам: сундуки, руды, столы, сломанные решётки,
блоки возрождения. Всё остальное — на совести карты. Поэтому:

- игроки могут ломать **только** решётки (`IRON_BARS`), руды (`*_ORE`,
  `INFESTED_*`) и блоки возрождения;
- ставить игроки могут **только** блок возрождения;
- взрывы, вода и лава не откатываются — не оставляйте их доступными.

---

## Выберите сценарий

| Ваш случай | Куда идти |
|---|---|
| Строю карту с нуля, полный контроль над геометрией | [Сценарий A](#сценарий-a--карта-с-нуля) — классическая разметка маркерами |
| Готовая скачанная карта с сотнями декоративных сундуков | [Сценарий B](#сценарий-b--готовая-карта-динамические-сундуки) — динамические сундуки |
| Нужна вторая арена по образцу уже работающей | [Сценарий C](#сценарий-c--клон-существующей-арены) — копирование папки |

Все три сценария сходятся на [общих шагах](#шаг-3-лут-обязательно) начиная с лута.

## Сценарий A — карта с нуля

Полная ручная разметка. Больше работы, зато точный контроль над тем, где именно
появляются сундуки и NPC.

### Шаг 1. Создать арену

Встаньте **в мире арены на месте будущего лобби** и выполните:

```
/escape create ТЮРЬМА 12
```

- `ТЮРЬМА` — ID. Приводится к верхнему регистру, кириллица допустима.
- `12` — максимум игроков (2–128). Необязательный, по умолчанию из
  `config.yml` → `defaults.max-players`.
- **Мир арены = мир, где вы стоите. Лобби = ваша текущая позиция.**

Числовые настройки берутся из `defaults` в `config.yml`. Дальше:

```
/escape setname ТЮРЬМА <green>Тюрьма Аркхем
/escape setdesc ТЮРЬМА Классическая карта на 12 человек
/escape setminplayers ТЮРЬМА 4
```

> Если позже вы переместите лобби командой `/escape setlobby`, **мир арены
> перепишется на текущий**. Выполнять её из другого мира — значит сломать арену:
> все точки останутся в старом мире. Валидатор этого не поймает.

### Шаг 2. Геометрия

Каждая `add*`-команда выдаёт в руку **предмет-маркер**. Точка записывается в
момент установки блока. Маркеры стакаются — берите пачку и расставляйте.

#### 2.1. Спавны — обязательно

```
/escape addspawn ТЮРЬМА
```

- Ставьте **не меньше, чем `max-players`**. Иначе точки переиспользуются по
  кругу и игроки заспавнятся друг на друге.
- Блок маяка **не ставится** — точка запоминается «в воздухе».
- Разносите точки: игроки появляются одновременно и сразу видят друг друга.

#### 2.2. Сундуки — обязательно (в сценарии A)

```
/escape addchest ТЮРЬМА
```

- Ставьте **больше точек, чем `chest-count`** (по умолчанию 75). Из всех точек
  на матч выбирается случайное подмножество — **именно избыток точек делает
  каждый матч непохожим на предыдущий.**
- **Сундуки должны быть одиночными.** Двойные не отслеживаются: LOOT-контракты,
  рефилл и «Волшебный ключик» их не увидят. Не ставьте точки вплотную.
- Блок сундука **не ставится** — плагин сам поставит его при старте матча и
  уберёт после.

Под каждым игровым сундуком плагин спавнит невидимую стойку — она носит подпись
«пусто / пополнен» и перехватывает клик. Это механика, а не баг.

#### 2.3. Руды — рекомендуется

```
/escape addore ТЮРЬМА
```

- **Блок ставится по-настоящему** (камень) — он часть карты.
- ⚠️ **Используются ВСЕ рудные точки, каждый матч.** Параметра `ore-count`
  не существует. Сколько точек поставили — столько руд и будет.
- Материал каждой руды выбирается случайно при старте:

| Материал | Шанс |
|---|---|
| Уголь | ~26 % |
| Железо | ~21 % |
| Заражённый камень (чешуйница!) | ~21 % |
| Золото | ~16 % |
| Алмаз | ~11 % |
| Изумруд | ~5 % |

- Руда **не даёт ни предметов, ни опыта** — она нужна для контрактов `MINE`/
  `BREAK`, темок, статистики и события «Подкоп».

#### 2.4. Точки NPC — рекомендуется

Сначала создайте тип NPC (см. [04-arena-content.md](04-arena-content.md#npc--торговцы-и-смотрящие)),
потом ставьте точки:

```
/escape createvillager БАРЫГА
/escape addvillager ТЮРЬМА БАРЫГА
```

- Тип должен существовать **до** `addvillager`.
- Точек ставьте больше, чем `trader-count` (по умолчанию 32) — по той же
  логике разнообразия.
- ⚠️ На матч попадает **случайное подмножество** точек. Если у типа-адресата
  COURIER-темки всего одна-две точки, он может не появиться в матче вообще, и
  темка станет невыполнимой. **Ставьте адресатам передачек по 4–5 точек.**

#### 2.5. Рычаги (именованные локации) — рекомендуется

```
/escape addlever ТЮРЬМА Столовая
/escape addlever ТЮРЬМА Прачечная второго этажа
```

- **Блок рычага ставится по-настоящему.**
- Имя = весь остаток строки, пробелы разрешены.
- Работает в двух ролях: цель контрактов и темок типа `ACTIVATE` **и** пункт в
  меню «Места» Личного проводника.
- Имя должно **посимвольно совпадать** с полем `idle` контракта/темки.

#### 2.6. Столы зачарований — опционально

```
/escape addtable ТЮРЬМА
```

Валюта зачарования — уровни XP (50 на старте, +10 за убийство) плюс лазурит,
который добывается из сундуков или покупается у торговцев.

#### 2.7. Точки финальной битвы — опционально

```
/escape addfinalspawn ТЮРЬМА
```

Если не задать ни одной, финальная битва использует обычные спавны. Отдельная
арена финала — способ сделать концовку зрелищной.

Переходите к [шагу 3](#шаг-3-лут-обязательно).

## Сценарий B — готовая карта (динамические сундуки)

Для скачанных карт, где сотни декоративных сундуков и расставлять маркер на
каждый — часы работы.

```
/escape create ТЮРЬМА 12
/escape set ТЮРЬМА dynamicchests 1
```

**Как это работает:** сундук карты, не отмеченный маркером, при **первом
открытии в идущем матче** получает лут по обычной схеме и становится
полноценным игровым — рефилл, LOOT-контракты, «Волшебный ключик», подпись.
Исходное содержимое запоминается и возвращается после матча.

### Условия и ограничения

| Условие | Детали |
|---|---|
| Сундук становится игровым при **любом** открытии | Включая случайное. Обратного пути в течение матча нет |
| Исходное содержимое живёт **только в памяти** | Краш сервера (не штатная остановка) = содержимое декоративных сундуков **потеряно безвозвратно** |
| Двойные сундуки не поддержаны | Как и везде |
| `chest-count` ничего не ограничивает | Число игровых сундуков растёт по мере открытия |

### ⚠️ Всё равно поставьте несколько точек сундуков

```
/escape addchest ТЮРЬМА
```

Предмет **«Молния прозрения»** раскладывается по сундукам **на старте матча**,
до того как хоть один динамический сундук открыт. Если точек сундуков нет
вообще, прозрения не попадут в мир — валидатор выдаст на это WARNING.

Достаточно 8–15 точек в проходных местах.

### Гибридный режим

Точки сундуков и `dynamicchests` **не исключают друг друга**. Рабочая схема:
маркерами разметить ключевые места (гарантированный лут на маршрутах), а
остальную карту отдать динамике.

Остальные шаги — как в сценарии A: [руды](#23-руды--рекомендуется),
[NPC](#24-точки-npc--рекомендуется), [рычаги](#25-рычаги-именованные-локации--рекомендуется),
[спавны](#21-спавны--обязательно) (спавны обязательны всегда).

## Сценарий C — клон существующей арены

Быстрый способ сделать вторую арену на той же карте или перенести настройки.

1. **Остановите сервер.** Копирование при работающем сервере будет затёрто.
2. Скопируйте папку:
   ```
   plugins/Escape/arenas/ТЮРЬМА  →  plugins/Escape/arenas/ТЮРЬМА2
   ```
   Имя папки — **обязательно в верхнем регистре**, иначе арена загрузится под
   ID в нижнем регистре и станет недоступна командам.
3. Откройте `arenas/ТЮРЬМА2/arena.yml` и поправьте:
   - `display-name` — иначе в меню будут две одинаковые строки;
   - `world` — если арена в другом мире;
   - `enabled: false` — включите после проверки.
4. Если мир другой — **`locations.yml` придётся переразметить**: координаты в
   нём привязаны к конкретному миру. Проще стереть содержимое списков и
   расставить точки заново.
5. Запустите сервер и выполните `/escape check ТЮРЬМА2`.

Глобальный контент (контракты, темки, типы NPC) копировать не нужно — он общий
для всех арен. Достаточно привязать контракты: `/escape addcontract ТЮРЬМА2 <CID>`.

## Шаг 3. Лут (обязательно)

Пустой пул лута — CRITICAL, арена не включится.

### Вариант 1 — командами

```
/escape additem ТЮРЬМА 50      ← предмет держим в руке
```

- Вес 1–250 — относительная частота. Вес 40 вдвое чаще, чем 20.
- Повторный вызов для того же предмета **меняет вес**, а не плодит дубликат.

### Вариант 2 — файлом (быстрее для больших пулов)

Остановите сервер, положите в `arenas/ТЮРЬМА/loot.yml`:

```yaml
items:
  - {type: WOODEN_PICKAXE, weight: 40}
  - {type: STONE_SWORD, weight: 18}
  - {type: ARROW, weight: 10, amount: 3}
```

Готовый «тюремный» пул на 40+ позиций — [`docs/examples/loot-prison.yml`](../examples/loot-prison.yml).

> Рукописный формат читается, но при **первом же сохранении арены** переписывается
> в сериализованный вид, а комментарии теряются. Держите исходник пула вне
> `plugins/`. Подробности — [02-configs.md](02-configs.md#потеря-комментариев-и-рукописного-формата).

### Сколько позиций

- Меньше 15 — WARNING: лут будет однообразным.
- 30–50 — комфортный разброс.
- Износ задавать не нужно: арена сама раскатывает случайную прочность
  40–90 % (`wearmin`/`wearmax`).

## Шаг 4. Контент

Контракты, NPC, темки — отдельный большой гайд:
**[04-arena-content.md](04-arena-content.md)**.

Минимум для запуска: **хотя бы один контракт**, привязанный к арене, иначе
WARNING и игрокам нечем заняться, кроме драки.

```
/escape createcontract КОПАТЕЛЬ
/escape contracttype КОПАТЕЛЬ MINE
/escape contractidle КОПАТЕЛЬ IRON_ORE
/escape contractdesc КОПАТЕЛЬ <white>Накопайте 3 железные руды
/escape contractamount КОПАТЕЛЬ 3
/escape contractprice КОПАТЕЛЬ 20
/escape addcontract ТЮРЬМА КОПАТЕЛЬ
```

## Шаг 5. Настроить мир

```
/escape worldsetup ТЮРЬМА
```

Выключает естественный спавн мобов, огонь, погоду, рейды, патрули, фантомов и
странствующих торговцев; включает мгновенный респаун. Команда идемпотентна —
запускайте после каждого пересоздания мира.

**Цикл дня и ночи сознательно не трогается** — он нужен событию «Кровавая луна»,
которое стартует только ночью.

---

## Шаг 6. Проверить и включить

```
/escape check ТЮРЬМА
```

Читайте отчёт сверху вниз:

| Метка | Что делать |
|---|---|
| **CRITICAL** | Обязательно чинить — арена не включится. К каждой строке приложена команда-подсказка |
| **WARNING** | Оценить осознанно. Часть предупреждений допустима (например, «зарплата не выдастся» при намеренно отключённой зарплате) |
| **GOOD** | Сводка: сколько точек, сундуков, NPC, контрактов. Сверьте с тем, что задумывали |

Когда CRITICAL = 0:

```
/escape enable ТЮРЬМА
```

`enable` прогоняет валидатор ещё раз и **откажет при любой CRITICAL-находке**.

## Шаг 7. Проверить в игре

Полный сценарий соло-теста — [06-operations.md](06-operations.md#соло-тест-арены).
Коротко:

```
/escape join ТЮРЬМА
/escape start ТЮРЬМА          ← стартует даже с одним игроком
/escape debug refill
/escape debug event FOG
/escape debug final
/escape stop ТЮРЬМА
```

После остановки **обойдите карту**: сундуки, руды и столы должны исчезнуть,
решётки — вернуться на место, торговцы и стойки — пропасть.

## Чек-лист готовности

**Обязательное (без этого не включится):**

- [ ] Арена создана, мир корректен
- [ ] Лобби задано
- [ ] Есть точки спавна (в идеале ≥ `max-players`)
- [ ] `min-players` ≤ `max-players`
- [ ] Пул лута не пуст
- [ ] Есть точки сундуков **или** включены `dynamicchests`
- [ ] Все типы NPC на точках существуют
- [ ] Все контракты и темки валидны
- [ ] `/escape check` → 0 CRITICAL

**Рекомендуемое:**

- [ ] Точек сундуков **больше**, чем `chest-count`
- [ ] Точек NPC **больше**, чем `trader-count`
- [ ] У адресатов COURIER-темок 4+ точки
- [ ] 15+ позиций в пуле лута
- [ ] Хотя бы 1 контракт привязан к арене
- [ ] Рудные точки расставлены (помните: используются **все**)
- [ ] Именованные рычаги для навигации и `ACTIVATE`-заданий
- [ ] `worldsetup` выполнен
- [ ] Соло-прогон пройден, мир после матча чист

## Типовые ошибки

| Симптом | Причина |
|---|---|
| «Мир не найден» в `check` | `world` в `arena.yml` указывает не туда — часто из-за `setlobby` из другого мира |
| Арена не видна игрокам | Не сделан `enable`, либо на ней уже идёт матч (`RUNNING`/`ENDING` скрывают арену из меню) |
| Игроки спавнятся друг на друге | Точек спавна меньше `max-players` |
| Сундук не рефиллится, LOOT не считается | Двойной сундук. Разнесите точки |
| Руд слишком много | Используются **все** точки — лишние надо удалять из `locations.yml` вручную |
| Ошибочную точку не убрать | Команд удаления нет. Правьте `locations.yml` при остановленном сервере |
| Правки конфига пропали | Затёрты автосохранением. См. [золотое правило](02-configs.md#золотое-правило-редактирования) |
