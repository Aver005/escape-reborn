# Эксплуатация: запуск, проверка, отладка, диагностика

Повседневная работа с уже настроенной ареной: тестирование, восстановление после
сбоев, разбор жалоб игроков.

---

## Ежедневные операции

| Задача | Команда | Побочные эффекты |
|---|---|---|
| Посмотреть состояние всех арен | `/escape list` | — |
| Проверить арену | `/escape check <ID>` | — |
| Открыть арену игрокам | `/escape enable <ID>` | Прогоняет валидатор, откажет при CRITICAL |
| Закрыть арену | `/escape disable <ID>` | **Идущий матч не останавливается** |
| Остановить матч | `/escape stop <ID>` | Отсчёт 5 сек, откат мира, возврат игроков |
| Остановить все матчи | `/escape stop all` | — |
| Принудительный старт | `/escape start <ID>` | Работает с одним игроком в лобби |
| Сохранить всё | `/escape save` | **Затирает ручные правки конфигов** |
| Перечитать конфиги | `/escape reload` | ⚠️ **Останавливает ВСЕ идущие матчи** |

> `/escape reload` — не безобидная команда. Она вызывает `stopAll()` до
> перезагрузки. Не используйте её на живом сервере в прайм-тайм.

### Порядок обновления конфигов

Безопасный порядок разобран в [02-configs.md](02-configs.md#золотое-правило-редактирования).
Коротко: **лучший способ — остановить сервер, править файлы, запустить.**

## Соло-тест арены

Полный прогон в одиночку. Требуются права `escape.admin` **и**
`escape.admin.debug`.

### 1. Войти и стартовать

```
/escape join ТЮРЬМА
/escape start ТЮРЬМА
```

`start` игнорирует `min-players` и запускает матч с одним игроком.

**Что проверить сразу:** вы телепортированы на точку спавна, в инвентаре
«Вилка», золото, «Личный проводник» и медный блок возрождения; на карте
появились сундуки, торговцы, руды и столы; идёт боссбар-таймер.

### 2. Проверить генерацию

Обойдите карту:

- сундуки стоят на размеченных точках и открываются;
- в сундуках 2–4 предмета, попадаются бумаги-контрактов;
- торговцы стоят, ПКМ открывает магазин / темки / меню выбора;
- руды на месте, ломаются, ничего не дропают;
- столы зачарований стоят и работают.

### 3. Прогнать механики

```
/escape debug refill          — рефилл всех сундуков, ответит числом
/escape debug contract        — довести бумагу из инвентаря до цели
/escape debug theme           — довести активную темку до цели
/escape debug gold 64         — выдать себе золота
/escape debug key             — «Волшебный ключик»
/escape debug insight         — «Молния прозрения»
```

### 4. Прогнать события

```
/escape debug event FOG
/escape debug event BLOOD_MOON     — работает и днём, canStart обходится
/escape debug event SEARCH
```

⚠️ Отладочный запуск **не печатает анонс** — вы увидите только эффекты. Это
нормально и отличается от естественного срабатывания.

### 5. Прогнать блок возрождения

1. Поставьте блок в укромном месте (два свободных блока сверху, твёрдая опора
   снизу) — должна ударить молния.
2. ПКМ по блоку — меню прокачки. `/escape debug gold 216` и прокачайте до
   алмаза.
3. `/escape debug death` — вы уходите в спектаторы на 5 секунд и возрождаетесь
   на блоке без вещей.
4. Сломайте свой блок — он должен вернуться в инвентарь предметом.

### 6. Финал и завершение

```
/escape debug glow      — фаза свечения
/escape debug final     — финальная битва (блоки возрождения аннулируются)
/escape debug finish    — завершить матч
```

Или `/escape stop ТЮРЬМА`.

> **Соло-матч сам не завершится.** Проверка «остался один выживший» срабатывает
> только в момент чьего-то выбывания. Пока вы живы — матч идёт.

### 7. Проверить откат мира

После остановки обойдите карту ещё раз:

- [ ] сундуки, руды и столы **исчезли**;
- [ ] сломанные решётки **вернулись**;
- [ ] торговцы и невидимые стойки **пропали**;
- [ ] выпавшие предметы **убраны**;
- [ ] ваш инвентарь **восстановлен** таким, каким был до входа;
- [ ] в логе сервера **нет стектрейсов**.

## Что видят игроки: чат и команды

### Каналы изолированы полностью

| Кто | Пишет и читает | Не видит |
|---|---|---|
| В лобби | Чат лобби | Всё остальное |
| Живой | Игровой чат + анонсы событий, зарплата, свечение | Реплики спектаторов |
| Мёртвый / спектатор | Чат спектаторов + сообщения о смертях | Реплики живых, **анонсы событий** |
| Вне сессии | Обычный глобальный чат | Всё, что происходит на арене |

Команды переключения канала **не существует** — маршрутизация автоматическая.

### Команды в матче заблокированы

Участнику сессии разрешены только: `/escape` без аргументов, `/escape leave`,
`/escape help`, `/escape stats`, `/escape info`.

Всё остальное отменяется — включая `/escape join` и команды других плагинов.
Обойти через `/minecraft:команда` нельзя, префикс нормализуется.

**Держатели `escape.admin` обходят фильтр полностью** — им доступны любые
команды прямо из матча.

## Диагностика проблем

### Арена

| Симптом | Причина | Решение |
|---|---|---|
| Арены нет в GUI | Не включена, либо идёт матч (`RUNNING`/`ENDING` скрывают арену) | `/escape enable`, либо дождаться конца матча |
| `enable` отказывает | Есть CRITICAL-находки | `/escape check <ID>`, чинить сверху вниз |
| «Мир не найден» | `world` в `arena.yml` указывает не туда | Часто следствие `setlobby` из другого мира |
| Вход отказан | Одно из шести условий `join` | Проверить: арена включена, лобби не полное, есть точки спавна, заданы лобби и мир |
| Игроки спавнятся вместе | Точек спавна меньше `max-players` | Добавить точки или снизить максимум |

### Контент

| Симптом | Причина | Решение |
|---|---|---|
| Товара нет в магазине | Неверный материал или отсутствующий `price` — запись выброшена **молча, без лога** | Проверить `traders.yml` |
| В магазине видно не все товары | Более 28 товаров у типа NPC — окно вмещает 28 | Разнести ассортимент по двум типам NPC. `/escape check` предупреждает |
| У NPC видно только 7 темок | Окно меню вмещает 7, сданные тоже занимают слоты | Разнести темки по нескольким типам NPC. `/escape check` предупреждает |
| Темка не сдаётся | `COURIER` без адресата; битый `idle`; адресат не заспавнился на этой карте | `/escape check`, добавить точек адресату |
| Контракт невыполним | Битый `idle` — валидатор ловит, но выдачу не блокирует | `/escape check`, исправить материал/рычаг |
| Контрактов нет в сундуках | Не привязаны к арене | `/escape addcontract <ID> <CID>` |
| Купленные вещи изношены | Износ применяется и к покупкам | Выключить износ или проставить урон в редакторе трейдов |

### Сундуки

| Симптом | Причина |
|---|---|
| Сундук не рефиллится | Двойной сундук — не поддерживается. Либо сундук не пуст: рефилл срабатывает **только для полностью пустого** |
| LOOT-контракт не считается | Двойной сундук |
| «Молний прозрения» нет в матче | Нет статических точек сундуков (арена только на `dynamicchests`), либо все сундуки при раздаче оказались полны |

### Игроки

| Симптом | Причина |
|---|---|
| Игрок потерял вещи | Проверьте `snapshots/<uuid>.yml`. Файл на месте = вещи восстановятся при следующем входе |
| Игрок вышел и «застрял» | Оффлайн-страж: 90 сек стоит зомби, 240 сек — окно возврата |
| Победа не объявлена | `/escape stop` завершает матч **без победителя и без MVP** — статистика не пишется |
| Статистика чужая | Поиск в `/escape stats` идёт **по нику**, не по UUID. При коллизии ников результат непредсказуем |

## Восстановление после краша

**Штатная остановка сервера** откатывает мир корректно — арены останавливаются
в `onDisable`.

**Краш или `kill -9`** — другое дело:

| Что теряется | Детали |
|---|---|
| **Откат мира** | Сундуки, руды, столы, стойки и торговцы останутся на карте. Решётки останутся сломанными |
| **Содержимое динамических сундуков** | Исходное содержимое живёт только в памяти — **потеряно безвозвратно** |
| **Блоки возрождения** | Состояние в памяти. Физические блоки останутся стоять на карте |
| Инвентари игроков | **Не теряются** — снапшоты лежат на диске и применяются при следующем входе игрока |

### Порядок восстановления

1. Запустите сервер, убедитесь, что плагин включился без ошибок.
2. Запустите и остановите матч на пострадавшей арене — генерация перезапишет
   сундуки/руды/столы на тех же точках, а `cleanup` их уберёт.
3. Оставшийся мусор (чужие стойки, торговцы, декоративные сундуки с потерянным
   содержимым) убирайте вручную.
4. Игроки восстановятся сами при входе — снапшоты одноразовые и удаляются после
   применения.

> Если карта критична, держите её бэкап и восстанавливайте мир из него — это
> быстрее ручной чистки.

## Известные ограничения

| Ограничение | Последствие |
|---|---|
| **Только одиночные сундуки** | Двойные не отслеживаются: рефилл, LOOT, ключик их не видят. Не ставьте точки вплотную |
| **Аренам нужен выделенный мир** | Защита от огня, распространения блоков и гниения листвы действует на весь мир — даже когда арена выключена |
| **Максимум 28 товаров и 7 темок на NPC** | Пагинации нет, лишнее не отображается. `/escape check` предупреждает |
| **Нет команд удаления точек** | Ошибочные точки убираются только правкой `locations.yml` |
| **Нет отвязки контракта от арены** | Убирается только правкой `contracts:` в `arena.yml` |
| **Нет удаления товара в GUI** | Только правкой `traders.yml` |
| **Команды только от игрока** | Из консоли/RCON/планировщика ничего не запустить |
| **Финальная битва без таймаута** | Останавливается только вручную |
| **Кредит убийства — 10 секунд** | Добивание позже не засчитывается как убийство |
| **Все рудные точки используются всегда** | Ограничивающего параметра нет |
| **ID контрактов/темок/NPC чувствительны к регистру** | Пишите заглавными — иначе сущность станет недоступна командам |
| **Запрет на рамки/картины следует за игроком** | Участник сессии не может ломать их даже вне арены |
| **Состояние матча не переживает рестарт** | См. раздел про краш |

Актуальный список дефектов — [`.memories/BUGS.md`](../../.memories/BUGS.md).

## Производительность

Точки, которые стоит держать в голове на нагруженном сервере:

- **Снапшот игрока пишется синхронно на диск** при каждом входе в лобби. При
  массовом заходе это I/O в главном потоке.
- **Смерть любого зомби на сервере** запускает перебор всех арен — на карте с
  мобофермами это лишняя работа.
- Один матч держит: боссбар, задачу раз в секунду, невидимую стойку под каждым
  сундуком (по умолчанию 75), villager'ов по числу `trader-count` (32) и
  отложенные задачи рефилла.
- Статистика пишется асинхронно — на главный поток не влияет.

## Права

| Право | По умолчанию | Даёт |
|---|---|---|
| `escape.admin` | `op` | Все админ-команды + обход блокировки команд в матче |
| `escape.admin.debug` | `op` | `/escape debug ...`. **Требуется вдобавок к `escape.admin`**, не наследуется |

Для отладочных команд нужны **оба** права одновременно.
