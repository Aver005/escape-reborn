# Справочник команд `/escape`

Полный список подкоманд с синтаксисом, диапазонами аргументов, правами и
побочными эффектами. Источник истины — [`EscapeCommand.java`](../../src/main/java/me/aver005/escape/command/EscapeCommand.java).

Задачные гайды («как сделать арену») — [03-arena-create.md](03-arena-create.md),
[04-arena-content.md](04-arena-content.md), [05-arena-tuning.md](05-arena-tuning.md).

---

## Общие правила

| Правило | Детали |
|---|---|
| Команда | `/escape`, алиасы `/es`, `/esc` |
| **Только для игрока** | Из консоли команда не работает вообще — отвечает «Только для игроков». Ни одна админ-операция из консоли/RCON/планировщика недоступна |
| Право админа | `escape.admin` (default: `op`) |
| Право отладки | `escape.admin.debug` (default: `op`) — **нужно вдобавок к** `escape.admin`, не наследуется |
| Без прав | Любая админ-подкоманда молча показывает справку игрока |
| ID арен/NPC/темок/контрактов | Автоматически переводятся в **ВЕРХНИЙ регистр**. `/escape check тюрьма` = `/escape check ТЮРЬМА`. Кириллица допустима |
| Tab-completion | Есть на всех уровнях: подкоманды, ID арен/NPC/темок/контрактов, ключи `set`, типы контрактов/темок, названия событий, ники игроков |
| Сохранение | Почти каждая изменяющая команда сразу пишет файл на диск (`ArenaManager.save` / `registry.save()`). Отдельный `/escape save` нужен редко |

> **Регистр ID — ловушка при ручном редактировании.** Папка арены на диске
> называется так же, как ID в верхнем регистре (`arenas/ТЮРЬМА/`). Если создать
> папку `arenas/тюрьма/` руками, арена загрузится под ID `тюрьма`, а команда
> `/escape check тюрьма` будет искать `ТЮРЬМА` и не найдёт её.

## Команды игрока

Доступны без прав.

| Команда | Действие |
|---|---|
| `/escape` | Открыть GUI выбора арены |
| `/escape join` | То же — GUI выбора арены |
| `/escape join <ID>` | Войти в лобби конкретной арены |
| `/escape leave` | Покинуть лобби/матч |
| `/escape stats [ник]` | Статистика игрока (по умолчанию — своя) |
| `/escape info` | Описание режима (список строк из `messages.yml` → `info`) |
| `/escape help` | Справка. Админам дополнительно показывается админский блок |

**Детали:**

- В GUI выбора арены попадают только арены с `enabled: true`, у которых **нет
  идущего матча** (фазы `RUNNING`/`ENDING` скрывают арену из списка). Арена в
  фазе `WAITING`/`COUNTDOWN` видна, войти можно.
- `/escape join <ID>` отказывает в шести случаях (`GameSession.join`): игрок уже
  в другой сессии, арена выключена, матч уже идёт (`RUNNING`/`ENDING`), лобби
  заполнено, лобби/мир не заданы, у арены нет ни одной точки спавна.
- `/escape stats` читает SQLite асинхронно — ответ приходит с задержкой в
  отдельном сообщении, это нормально.
- `/escape leave` вне матча отвечает ошибкой `errors.not-in-game`.

## Админ: управление плагином

| Команда | Действие | Осторожно |
|---|---|---|
| `/escape save` | Записать на диск все арены, контракты, темки, торговцев | Затирает ручные правки этих файлов, сделанные при работающем сервере |
| `/escape reload` | **Остановить все матчи** (`stopAll`), перечитать config.yml, messages.yml, арены, контракты, темки, торговцев | Прерывает идущие игры без предупреждения |
| `/escape list` | Список всех арен: статус (выключена/ожидание/отсчёт/идёт), текущее число игроков, максимум | — |

**Порядок при ручном редактировании конфигов** — см. [02-configs.md](02-configs.md#золотое-правило-редактирования),
там же разобрана ловушка «правки затёрлись при выключении сервера».

## Админ: жизненный цикл арены

| Команда | Аргументы | Действие |
|---|---|---|
| `/escape create <ID> [макс]` | `макс`: 2–128 | Создать арену. **Мир = текущий мир админа, лобби = текущая позиция.** Числовые настройки берутся из `defaults` в config.yml |
| `/escape remove <ID>` | — | Удалить арену: останавливает матч и **рекурсивно удаляет папку** `arenas/<ID>/`. Без подтверждения, без корзины |
| `/escape enable <ID>` | — | Прогнать валидатор и включить арену. **CRITICAL-находки блокируют включение** |
| `/escape disable <ID>` | — | Выключить (скрыть из меню). Идущий матч не останавливается |
| `/escape start <ID>` | — | Принудительный старт матча (`forceStart`). Работает даже с **одним** игроком в лобби — основной инструмент соло-теста |
| `/escape stop <ID>` | — | Остановить матч на арене |
| `/escape stop all` | — | Остановить матчи на **всех** аренах |
| `/escape setlobby <ID>` | — | Лобби = позиция админа (центр блока, `+0.5` по X/Z). Заодно **переписывает мир арены** на текущий |
| `/escape setname <ID> <имя...>` | остаток строки | Отображаемое имя в меню. MiniMessage: `<green>Тюрьма Аркхем` |
| `/escape setdesc <ID> <текст...>` | остаток строки | Описание в меню. MiniMessage |
| `/escape setmaxplayers <ID> <N>` | 1–128 | Максимум игроков |
| `/escape setminplayers <ID> <N>` | 1–128 | Минимум для автостарта |

**Детали:**

- `create` не проверяет корректность мира — просто берёт мир, где стоит админ.
  Если позже мир переименуют/удалят, `check` выдаст CRITICAL «мир не найден».
- `setlobby` **меняет и мир арены**. Если вы телепортировались в другой мир и
  выполнили `setlobby`, арена «переедет» туда, а все точки останутся в старом
  мире — арена сломается. Валидатор этого не ловит.
- `enable` при CRITICAL показывает отчёт и не включает арену. `disable` никаких
  проверок не делает.
- `setminplayers > setmaxplayers` — команда пройдёт, но `check` даст CRITICAL.

## Админ: геометрия арены (маркеры)

Все `add*`-команды выдают в инвентарь **предмет-маркер**. Точка записывается в
арену в момент **установки блока** ([`SetupListener.java`](../../src/main/java/me/aver005/escape/listener/SetupListener.java)),
а не в момент выдачи предмета. Один маркер = одна точка; на стак маркеров можно
расставить весь стак.

| Команда | Предмет-маркер | Что добавляет | Блок-подсказка на карте |
|---|---|---|---|
| `/escape addspawn <ID>` | Маяк | Точка спавна игрока | 🩶 два серых стекла |
| `/escape addfinalspawn <ID>` | Лоудстоун | Точка спавна финальной битвы | 🖤 два чёрных стекла |
| `/escape addchest <ID>` | Сундук | Точка игрового сундука | 🟫 сундук |
| `/escape addtable <ID>` | Стол зачарований | Точка стола зачарований | ❌ нет |
| `/escape addore <ID>` | Камень | Рудная точка | ✅ сам камень (маркер тратится) |
| `/escape addlever <ID> <имя...>` | Рычаг | Именованный рычаг | ✅ сам рычаг (маркер тратится) |
| `/escape addvillager <ID> <VID>` | Верстак | Точка NPC типа `VID` | 🩷 два розовых стекла |

### Блоки-подсказки точек

Чтобы при настройке было видно, где что уже размещено, точки спавнов, финальных
спавнов, NPC и сундуков помечаются блоками
([`SetupMarkers.java`](../../src/main/java/me/aver005/escape/arena/SetupMarkers.java)):

- **Стекло** (серое — спавн, чёрное — финальный спавн, розовое — NPC) ставится
  двумя блоками — ровно там, где будет стоять фигура.
- **Сундук** точки сундука остаётся настоящим блоком. Его содержимое игрой
  **не используется и чистится** — и на старте матча, и после него.
- **Сломать подсказку = удалить точку.** Достаточно сломать один блок стекла из
  двух; остаток убирается сам, изменение сразу пишется в `locations.yml`.
  Ломать может только игрок с `escape.admin` и только когда арена не в игре.
- **На матч подсказки снимаются** (до телепорта игроков, чтобы никто не
  появился внутри стекла) и **возвращаются после очистки мира**.
- `/escape markers <ID>` — расставить подсказки всех точек заново (например,
  для арен, настроенных до этой фичи, или если блоки кто-то затёр). Работает
  только когда арена не в игре; чужие блоки карты не затирает — маркер встаёт
  только в воздух/заменяемый блок.
- Точки столов, руд и рычагов подсказок не имеют: руда и рычаг ставятся самим
  маркером как настоящий блок, стол — точка без визуализации.

**Детали и условия:**

- **Маркер работает у любого админа и на любой арене** — привязка (арена, тип,
  доп. значение) лежит в PDC предмета. Предмет можно передать другому админу,
  он поставит точки в ту же арену. У игрока без `escape.admin` установка
  отменяется.
- `addlever <ID> <имя>` — имя обязательно, берётся весь остаток строки
  (пробелы разрешены: `/escape addlever ТЮРЬМА Столовая второго этажа`). Это имя
  используется двумя способами: цель контрактов/темок типа `ACTIVATE` и пункт в
  меню «Локации» Личного проводника.
- `addvillager <ID> <VID>` — тип NPC должен **уже существовать** в `traders.yml`,
  иначе команда откажет (`errors.trader-not-exists`).
- `addore` и `addlever` ставят настоящий блок — он останется на карте как часть
  декорации. Остальные маркеры блок не ставят, точка запоминается «в воздухе».
- Сундуки на точках должны быть **одиночными**. Двойной сундук не отслеживается
  (LOOT-контракты, рефилл, «Волшебный ключик» его не увидят) — точки сундуков
  не ставьте вплотную друг к другу.

> ⚠️ **Команд удаления точек нет.** Ни `removespawn`, ни очистки списка. Чтобы
> убрать ошибочно поставленную точку, нужно править
> `arenas/<ID>/locations.yml` вручную — порядок безопасного редактирования в
> [02-configs.md](02-configs.md#золотое-правило-редактирования).

## Админ: числовые настройки арены

```
/escape set <ID> <ключ> <значение>
```

Минимум для всех ключей — **1**, кроме `wearmin`, `wearmax`, `dynamicchests`
(минимум 0). Максимум — `Integer.MAX_VALUE`, но `wearmin`/`wearmax`
дополнительно обрезаются до **99**.

| Ключ | Ключ в `arena.yml` | Дефолт | Смысл |
|---|---|---|---|
| `duration` | `duration-seconds` | 1200 | Длительность до финальной битвы, сек |
| `eventinterval` | `event-interval-seconds` | 210 | Интервал случайных событий, сек |
| `salaryinterval` | `salary-interval-seconds` | 600 | Интервал «зарплаты», сек |
| `salarygold` | `salary-gold` | 16 | Золота за одну зарплату |
| `glowtime` | `glow-seconds-before-end` | 600 | За сколько секунд до конца включается свечение |
| `glowgold` | `glow-bonus-gold` | 18 | Бонусное золото при включении свечения |
| `chests` | `chest-count` | 75 | Сколько сундуков активируется на матч |
| `traders` | `trader-count` | 32 | Сколько NPC спавнится на матч |
| `tables` | `table-count` | 5 | Сколько столов зачарований ставится |
| `forkuses` | `fork-uses` | 4 | Прочность «Вилки» — сколько решёток ломает |
| `startgold` | `start-gold` | 24 | Стартовое золото игрока |
| `startdelay` | `start-delay-seconds` | 60 | Отсчёт при наборе минимума, сек |
| `startdelayfull` | `start-delay-full-seconds` | 10 | Отсчёт при полной арене, сек |
| `wearmin` | `wear-min-percent` | 40 | Нижняя граница случайного износа лута, % (0 = выключить) |
| `wearmax` | `wear-max-percent` | 90 | Верхняя граница износа, % |
| `dynamicchests` | `dynamic-chests` | 0 | `1` — немаркированные сундуки карты становятся игровыми |

**Детали:**

- `dynamicchests` — булев параметр, передаётся числом: `1` = включено, `0` = выключено.
- Ноль запрещён для `salarygold`, `startgold`, `glowgold` и всех интервалов.
  Чтобы «выключить» зарплату или события, задайте интервал **больше
  длительности матча** (валидатор выдаст WARNING — это ожидаемо).
- `chests`/`traders`/`tables` — это **сколько взять из доступных точек**, а не
  сколько точек существует. Если значение больше числа точек, `check` выдаст
  WARNING; фактически будет использовано столько, сколько есть.
- Разбор влияния каждого параметра на игру и готовые пресеты — [05-arena-tuning.md](05-arena-tuning.md).

## Админ: восстановление блоков-подсказок

```
/escape markers <ID>
```

Проходит по всем точкам арены и расставляет блоки-подсказки: серые стёкла на
спавнах, чёрные на финальных спавнах, розовые на точках NPC, сундуки на точках
сундуков. Нужна, если арена настраивалась до появления подсказок или блоки были
затёрты.

- Работает, **только когда арена не в игре** (иначе `admin.markers-busy`) — иначе
  подсказки залезли бы в идущий матч.
- Идемпотентна: повторный запуск ничего не ломает.
- **Чужие блоки не затирает** — маркер встаёт только в воздух или заменяемый
  блок; занятая точка просто пропускается.
- Содержимое сундуков-точек чистится.

## Админ: gamerules мира арены

```
/escape worldsetup <ID>
```

Приводит мир арены в «матчевое» состояние. Идемпотентна — запускайте после
каждого пересоздания мира.

| Gamerule | Значение | Зачем |
|---|---|---|
| `spawn_mobs` | `false` | Никакого естественного спавна (наши NPC и стражи спавнятся плагином — их это не касается) |
| `fire_spread_radius_around_player` | `0` | Огонь не распространяется |
| `mob_griefing` | `false` | Криперы/эндермены не портят арену |
| `advance_weather` | `false` | Без дождя (плюс сразу гасится текущая погода) |
| `spawn_phantoms` | `false` | Фантомы не мешают |
| `spawn_wandering_traders` | `false` | Странствующий торговец ≠ наш торговец |
| `spawn_patrols` | `false` | Патрули не мешают |
| `raids` | `false` | Рейды выключены |
| `show_advancement_messages` | `false` | Без спама достижений |
| `spectators_generate_chunks` | `false` | Спектаторы не генерируют чанки |
| `immediate_respawn` | `true` | Страховка: смерти фейковые, экран смерти не нужен |

**Сознательно НЕ трогается:** `advance_time` (цикл дня и ночи нужен событию
«Кровавая луна»), `natural_health_regeneration`, `random_tick_speed`.

## Админ: лут арены

| Команда | Аргументы | Действие |
|---|---|---|
| `/escape additem <ID> <вес>` | вес 1–250 | Добавить предмет **из главной руки** в пул лута с указанным весом |
| `/escape edititems <ID>` | — | Открыть GUI-редактор пула (54 слота) |

**Детали:**

- `additem` **заменяет** запись, если такой же предмет уже есть в пуле
  (`isSimilar`), — повторный вызов меняет вес, а не плодит дубликаты.
- Вес — относительная частота в рулетке. Предмет с весом 40 выпадает вдвое чаще,
  чем с весом 20. Абсолютные значения не важны, важны пропорции.
- GUI-редактор: пул раскладывается по слотам, у каждого предмета в лоре показан
  вес. Можно свободно доставать/класть предметы. **Пул пересобирается при
  закрытии инвентаря.** Существующие предметы сохраняют свой вес, **новые
  получают вес 1** — точный вес задавайте через `additem`.
- Больше 54 позиций в пуле GUI не покажет (лишние не отобразятся, но при
  закрытии редактора будут потеряны — правьте такой пул через `loot.yml`).
- Износ предметов задавать руками не нужно — арена сама раскатывает случайную
  прочность в диапазоне `wearmin`–`wearmax`.

## Админ: касты (стартовые наборы)

Каст = доп. предметы + стартовое золото на выбор игрока (системные предметы
выдаются всегда). Игрок выбирает каст в лобби через GUI. Глобальные шаблоны —
`kits.yml`; на арену ставится копией (правится независимо). Подробности —
[04-arena-content.md](04-arena-content.md#касты-стартовые-наборы), дизайн —
`docs/07-decisions` §13.

| Команда | Аргументы | Действие |
|---|---|---|
| `/escape kit list [ID]` | — | Глобальные шаблоны или касты арены `ID` |
| `/escape kit copy <Каст> <ID> [НовыйId]` | — | Копия глобального каста на арену |
| `/escape kit create <ID> <KID>` | — | Пустой каст арены |
| `/escape kit delete <ID> <KID>` | — | Удалить каст арены |
| `/escape kit edit <ID> <KID>` | — | GUI-редактор предметов каста (как редактор лута) |
| `/escape kit name <ID> <KID> <Имя...>` | MiniMessage | Имя каста в меню |
| `/escape kit icon <ID> <KID>` | из руки | Иконка каста = предмет в главной руке |
| `/escape kit gold <ID> <KID> <N>` | -1..100000 | Стартовое золото каста (-1 = `start-gold` арены) |
| `/escape kit default <ID> <random\|none\|KID>` | — | Каст для не выбравших в лобби |

**Детали:**

- Всё снаряжение кастов принято выдавать **почти сломанным** (`damage` в спеке
  предмета) — ремонт снаряжения есть двигатель игры.
- GUI-редактор предметов работает как редактор лута: разложил → пересобралось
  при закрытии. Имя/иконку/золото правят командами, не в GUI.
- Броня в предметах каста авто-надевается в слоты при выдаче.
- Нет ни одного каста на арене → всем выдаётся одинаково (как раньше),
  селектор в лобби не появляется.

Наполнение пула и готовый пример — [04-arena-content.md](04-arena-content.md#лут-сундуков).

## Админ: NPC (торговцы и смотрящие)

Типы NPC глобальны — один тип можно ставить на разные арены.

| Команда | Аргументы | Действие |
|---|---|---|
| `/escape createvillager <VID>` | — | Создать тип NPC |
| `/escape villagername <VID> <имя...>` | остаток строки | Отображаемое имя (MiniMessage) |
| `/escape addtrade <VID>` | — | GUI добавления товара в ассортимент |
| `/escape addtheme <VID> <TID>` | — | Привязать темку к NPC (NPC с темками = **смотрящий**) |
| `/escape removetheme <VID> <TID>` | — | Отвязать темку |
| `/escape addscrap <VID> <цена>` | 1–100000 | **Мусорщик**: скупать предмет **из руки** (база; цена = база × износ) |
| `/escape removescrap <VID>` | из руки | Убрать предмет из руки из прайса Мусорщика |
| `/escape scrapwear <VID> <%>` | 0–99 | Мин. износ, с которого Мусорщик берёт вещь (дефолт 25) |
| `/escape scraplist <VID>` | — | Показать прайс Мусорщика |
| `/escape addvillager <ID> <VID>` | — | Поставить точку NPC на арену (см. маркеры) |

**Детали:**

- NPC = магазин (трейды) и/или смотрящий (темки) и/или **Мусорщик** (scrap).
  Несколько ролей → при ПКМ меню выбора «Магазин / Темки / Мусорщик».
- **Мусорщик** скупает только **сломанное** снаряжение с прочностью (износ ≥
  `scrapwear`), цена растёт с износом. Системку (вилка/проводник/блок/контракты/
  темки) продать нельзя. Подробнее — [04-arena-content.md](04-arena-content.md#мусорщик-скупка-сломанного)
  и `docs/07-decisions` §14.
- NPC **вообще без ролей**, поставленный на арену, даёт WARNING в `check`.
- `addtheme` требует, чтобы темка уже существовала; `removetheme` работает
  всегда (несуществующий ID просто игнорируется).

|---|---|---|
| `/escape createcontract <CID>` | — | Создать контракт |
| `/escape contracttype <CID> <тип>` | см. ниже | Тип задания |

## Админ: проверка арены

```
/escape check <ID>
```

Прогоняет валидатор ([`ArenaCheck.java`](../../src/main/java/me/aver005/escape/arena/ArenaCheck.java))
и печатает отчёт: строки CRITICAL / WARNING / GOOD, к каждой проблеме —
команда-подсказка, в конце — счётчик проблем.

**Тот же валидатор автоматически запускается при `/escape enable`, и любая
CRITICAL-находка блокирует включение арены.**

### Что считается CRITICAL (арена не включится)

| Находка | Причина |
|---|---|
| Мир не найден | `world` в `arena.yml` указывает на незагруженный/несуществующий мир |
| Не задано лобби | `/escape setlobby <ID>` |
| Нет точек спавна | `/escape addspawn <ID>` |
| `min-players` > `max-players` | Матч никогда не стартует |
| Пустой пул лута | `/escape additem` или заполнить `loot.yml` |
| Нет точек сундуков **и** выключены динамические сундуки | Лут негде выдавать |
| Тип NPC с точки отсутствует в `traders.yml` | Опечатка в `addvillager` или удалённый тип |
| Темка, привязанная к NPC, не существует | Удалили темку, но не отвязали |
| У темки не задан тип | `/escape themetype` |
| У темки `ACTIVATE` цель — не существующий на арене рычаг | Имя рычага не совпадает |
| У темки `MINE`/`BREAK`/`FIND`/`DELIVERY` цель — не материал | Опечатка в названии материала |
| У темки `COURIER` адресат не существует | `/escape themereturn <TID> <VID>` |
| Контракт арены не существует | Удалили контракт, но не убрали из `arena.yml` |
| У контракта не задан тип | `/escape contracttype` |
| У контракта `ACTIVATE` цель — не существующий рычаг | Имя рычага не совпадает |
| У контракта `MINE`/`BREAK`/`FIND` цель — не материал | Опечатка |

### Что считается WARNING (включится, но подозрительно)

| Находка | Смысл |
|---|---|
| Точек спавна меньше `max-players` | Точки будут переиспользованы, игроки заспавнятся вместе |
| В пуле лута меньше 15 позиций | Лут будет однообразным |
| Нет точек сундуков при включённых динамических | «Молнию прозрения» некуда положить |
| `chest-count` > числа точек сундуков | Возьмётся столько, сколько есть |
| `trader-count` > числа точек NPC | То же |
| `table-count` > числа точек столов | То же |
| `glow-seconds-before-end` ≥ половины длительности | Свечение включится слишком рано |
| `salary-interval-seconds` ≥ длительности | Зарплата не выдастся ни разу |
| `event-interval-seconds` ≥ длительности | События не произойдут ни разу |
| NPC на арене без трейдов и без темок | Бесполезный NPC |
| У NPC больше 28 товаров | В окно магазина влезает 28, лишние не покажутся |
| У NPC больше 7 темок | В меню влезает 7, лишние не покажутся |
| У арены нет ни одного контракта | Игрокам нечего выполнять |

> Валидатор проверяет темки **только у тех типов NPC, которые фактически
> расставлены на этой арене**. Темки у неиспользуемых типов остаются
> непроверенными.

## Админ: отладка и соло-тест

```
/escape debug <действие> [аргумент]
```

**Условия запуска (все три обязательны):**

1. право `escape.admin` **и** `escape.admin.debug`;
2. вы находитесь в сессии арены;
3. матч в фазе `RUNNING` (не лобби, не отсчёт).

| Действие | Аргумент | Что делает |
|---|---|---|
| `death` | — | Полный цикл собственной смерти (дроп инвентаря + обработка) |
| `kill <ник>` | ник онлайн-игрока в матче | То же для другого игрока |
| `refill` | — | Принудительный рефилл **всех** сундуков, отвечает числом обновлённых |
| `contract` | — | Довести до цели контракт-бумагу из инвентаря |
| `theme` | — | Довести до цели активную темку |
| `event <ТИП>` | имя события | Запустить событие **в обход проверки условий** — Кровавую луну можно вызвать днём |
| `glow` | — | Включить фазу свечения |
| `final` | — | Начать финальную битву |
| `finish` | — | Завершить матч |
| `gold <N>` | 1–1024 | Выдать себе золото |
| `key` | — | Выдать «Волшебный ключик» |
| `insight` | — | Выдать «Молнию прозрения» |

**Соло-тест:** `/escape start <ID>` стартует матч с одним игроком. Проверка
«остался один выживший» срабатывает только в момент чьего-то выбывания, поэтому
соло-матч сам не завершится — он идёт, пока соло-игрок не выбыл окончательно.
Смерть при живом блоке возрождения матч не заканчивает.

Пошаговый сценарий соло-прогона — [06-operations.md](06-operations.md#соло-тест-арены).

## Админ: продвинутое логирование

```
/escape debuglog [on|off|save|clear|status]
```

Право — `escape.admin` (отдельного права нет, `escape.admin.debug` не требуется).
Команда игрока: из консоли сервера, как и все остальные команды плагина, не работает.

Включённое логирование пишет **каждое игровое действие** — вход/выход, старт и
генерацию арены, бой и кредит убийце, смерть и выбывание, лут и рефилл сундуков,
контракты и темки, блоки возрождения, оффлайн-стражей, события, покупки,
изменения блоков и откат мира — сразу в два места:

- **в консоль сервера** с префиксом `[DBG][КАТЕГОРИЯ]` (попадает и в `logs/latest.log`);
- **в собственный кольцевой буфер** в памяти, который сохраняется на диск отдельной
  командой.

| Действие | Что делает |
|---|---|
| (без аргумента) или `status` | Состояние: вкл/выкл, дублирование в консоль, заполнение буфера |
| `on` / `off` | Включить/выключить. Состояние пишется в `config.yml` → `debug-log.enabled` и переживает рестарт |
| `save` | Записать буфер в `plugins/Escape/logs/debug-<дата>-<время>.log`, ответ — имя файла и число строк |
| `clear` | Очистить буфер (файлы, сохранённые ранее, не трогает) |

**Формат строки:** `20:45:01.123 [RESPAWN] upgrade player=Aver005 COPPER->IRON charges=3 at=prison:120,64,-33`

Категории для грепа: `SESSION`, `PLAYER`, `COMBAT`, `CHEST`, `CONTRACT`, `THEME`,
`RESPAWN`, `GUARD`, `EVENT`, `WORLD`, `SHOP`, `MECH`, `ADMIN`.

**Что важно знать:**

- `off` буфер **не очищает** — выключил, потом сохранил, всё на месте.
- Буфер кольцевой: при переполнении (`debug-log.buffer-lines`, по умолчанию 20000)
  старые строки вытесняются, `status` показывает счётчик вытесненных. Полная
  история при этом остаётся в `logs/latest.log` сервера, если включён вывод в консоль.
- Строки логов всегда ASCII: кириллица (ник, название мира) заменяется на `?` —
  иначе Windows-консоль превращает вывод в кашу.
- Выключенное логирование не стоит ничего: строки не форматируются.
- Это диагностика, а не постоянный режим — на людном матче консоль забивается
  быстро. Включили → сыграли → `save` → `off`.

## Ошибки команд

Тексты берутся из `messages.yml` → `errors.*`.

| Ключ | Когда |
|---|---|
| `errors.not-enough-args` | Не хватает обязательного аргумента |
| `errors.unknown-subcommand` | Неизвестная подкоманда или ключ `set` |
| `errors.must-be-number` | Аргумент не число |
| `errors.number-out-of-range` | Число вне допустимого диапазона |
| `errors.arena-not-exists` / `errors.arena-exists` | Проблема с ID арены |
| `errors.contract-not-exists` / `errors.contract-exists` | Проблема с ID контракта |
| `errors.theme-not-exists` / `errors.theme-exists` | Проблема с ID темки |
| `errors.trader-not-exists` / `errors.trader-exists` | Проблема с ID типа NPC |
| `errors.wrong-contract-type` / `errors.wrong-theme-type` | Неизвестный тип |
| `errors.no-item-in-hand` | `additem` с пустой рукой |
| `errors.world-not-loaded` | `worldsetup` для арены в незагруженном мире |
| `errors.not-in-game` | `/escape leave` вне матча |
