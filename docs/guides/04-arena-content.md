# Наполнение арены контентом

Лут, контракты, темки, NPC, руды, рычаги — что из чего состоит, как связано и
какие есть жёсткие ограничения.

Создание арены — [03-arena-create.md](03-arena-create.md).
Балансировка чисел — [05-arena-tuning.md](05-arena-tuning.md).

---

## Три слоя контента

| Слой | Где живёт | Область | Как попадает в матч |
|---|---|---|---|
| **Лут** | `arenas/<ID>/loot.yml` | Одна арена | Сундуки, награда события «Взятка», покупки у NPC |
| **Контракты** | `contracts.yml` (глобально) | Привязка списком в `arena.yml` | Бумаги в сундуках |
| **Темки** | `themes.yml` (глобально) | Привязка к типам NPC | Смотрящие выдают лично |
| **NPC** | `traders.yml` (глобально) | Точки на арене | Спавнятся при старте матча |

Глобальные сущности переиспользуются между аренами — один «Барыга» работает на
всех картах. Лут привязан к конкретной арене.

## Лут сундуков

### Как наполняется сундук

При старте матча (и при каждом рефилле) в сундук кладётся **2–4 предмета**.
Слоты выбираются случайно из **27** — это ещё одна причина, почему сундуки
должны быть одиночными.

Для каждого слота бросается кубик: **25 % — бумага-контракт**, иначе — предмет
из пула лута по весам.

> Если у арены нет валидных контрактов, все 2–4 слота заполняются лутом.

### Веса

```yaml
items:
  - {type: WOODEN_PICKAXE, weight: 40}   # частый хлам
  - {type: STONE_SWORD, weight: 18}      # обычное
  - {type: IRON_CHESTPLATE, weight: 2}   # редкость
```

Вес — доля в рулетке. Абсолютные значения не важны, важны пропорции: `40` против
`2` = в 20 раз чаще. Минимум 1, максимум 250 (через `additem`).

### Износ — не задавайте прочность руками

Каждому выданному предмету арена раскатывает **случайный износ** в диапазоне
`wearmin`–`wearmax` (по умолчанию 40–90 %). Так одна и та же каменная лопата
каждый раз ломана по-разному.

| Условие | Поведение |
|---|---|
| `wearmax = 0` | Износ выключен полностью |
| Предмет без прочности | Не трогается |
| **У предмета уже проставлен урон** | **Не трогается** — способ выдавать целые вещи |
| `wearmin > wearmax` | Диапазон вырождается в одно значение, ошибки нет |

⚠️ **Износ применяется и к покупкам у торговцев.** Купленная железная кирка
придёт изношенной на 40–90 %. Чтобы продавать целое — либо выключите износ на
арене, либо кладите в редактор трейдов предмет с уже выставленным уроном.

### Что стоит и не стоит класть в пул

- **Обязательно** — кирки уровня железа и выше. Без них изумрудные блоки
  возрождения почти невозможно сломать за разумное время.
- **Обязательно** — лазурит, если на арене есть столы зачарований (иначе
  уровни XP некуда девать).
- **Не нужно** — золотые слитки: это валюта, она приходит из старта, зарплаты,
  свечения и наград.
- **Не нужно** — блоки для строительства: ставить блоки игрокам запрещено.

Готовый «тюремный» пул на 40+ позиций — [`docs/examples/loot-prison.yml`](../examples/loot-prison.yml).

## Контракты

Задания, которые **не надо никому сдавать** — награда падает сразу при
достижении цели.

### Шесть типов

| Тип | Поле `idle` | Условие выполнения |
|---|---|---|
| `KILLS` | не используется | Убить N игроков. Засчитывается и убийство оффлайн-стража |
| `ACTIVATE` | **имя рычага** | Дёрнуть именованный рычаг N раз |
| `MINE` | **материал** | Сломать N блоков этого материала |
| `BREAK` | **материал** | То же самое — обрабатывается тем же событием, что и `MINE` |
| `FIND` | **материал** | **Иметь при себе** N предметов. Не изымаются |
| `LOOT` | не используется | Впервые открыть N сундуков (повторные не считаются) |

`MINE` и `BREAK` в текущей реализации эквивалентны — разница только
в семантике названия для игрока.

### Как контракт попадает в игру

1. Создан в `contracts.yml` и имеет **тип** и `amount > 0`.
2. **Привязан к арене** — `/escape addcontract <ID> <CID>`. Без этого шага
   контракт не появится в матче никогда.
3. Выпал бумагой из сундука (25 % на слот).

### Бумага-контракт — важные детали

- Предмет — обычная бумага. **Прогресс хранится в самой бумаге.**
- Бумаги **не стакаются** (у каждой свой скрытый идентификатор) — у игрока может
  быть несколько разных бумаг одновременно.
- При смерти бумага **выпадает вместе с накопленным прогрессом**. Подобравший
  продолжает с того же места — это фишка оригинала, а не баг.
- За одно действие продвигается **только одна бумага**. Две одинаковые бумаги
  параллельно не качаются.
- `FIND`: прогресс запоминает **максимум** (выбросил предметы — счётчик не
  откатится), но завершение проверяется по **фактическому** количеству в
  инвентаре в текущий момент.

### Пример

```
/escape createcontract КОПАТЕЛЬ
/escape contracttype КОПАТЕЛЬ MINE
/escape contractidle КОПАТЕЛЬ IRON_ORE
/escape contractdesc КОПАТЕЛЬ <white>Накопайте 3 железные руды
/escape contractamount КОПАТЕЛЬ 3
/escape contractprice КОПАТЕЛЬ 20
/escape addcontract ТЮРЬМА КОПАТЕЛЬ
```

> ⚠️ Битый `idle` **не мешает контракту выдаваться**. Контракт `MINE` с
> несуществующим материалом попадёт в сундуки и будет невыполним. Ловит это
> только `/escape check`.

## Темки

Задания смотрящих. В отличие от контракта, темку надо **сдать NPC**.

### Восемь типов

Те же шесть, что у контрактов, плюс два:

| Тип | Поле `idle` | Условие сдачи |
|---|---|---|
| `DELIVERY` | **материал** | Принести N предметов. **Изымаются при сдаче** |
| `COURIER` | не используется | Донести «Передачку» указанному адресату |

### Правила механики

| Правило | Детали |
|---|---|
| Одна активная темка | Взять вторую нельзя, пока не сдана или не брошена текущая |
| Одна сдача за матч | Сданная темка помечается серой и больше не выдаётся **этому игроку** |
| Кулдаун 60 сек | Начисляется **только при броске**, не при сдаче. После штатной сдачи новую темку можно брать сразу |
| Награда | `gold` темки + **10 % шанс** на «Волшебный ключик» |
| Сдача перехватывает клик | Подошли к NPC с готовой темкой — она сдастся, **магазин не откроется**. Кликните второй раз |

### Кому сдавать — поле `turn-in`

| Значение | Смысл | Тонкость |
|---|---|---|
| `SELF` | Тому же жителю, что выдал | Сверка по **UUID конкретной сущности**, а не по типу. Другой NPC того же типа темку не примет — а игрок не знает, какой именно из десятка одинаковых «Смотрящих» её выдал |
| `ANY` | Любому смотрящему | Принимающий должен иметь **хотя бы одну** темку в списке. Чистый торговец не примет |
| `<VID>` | Конкретному типу NPC | Требования «быть смотрящим» **нет** — адресатом может быть чистый торговец |

### `COURIER` — передачки

При взятии игрок получает **предмет-свёрток**. Механика намеренно рискованная:

- Свёрток **выпадает при смерти**.
- **Сдать его может кто угодно**, кто подобрал — награда достаётся
  доставившему.
- У исходного владельца темка при этом закрывается **без награды**.
- Доставивший чужой свёрток теряет возможность взять эту темку самому в этом
  матче.

⚠️ `COURIER` с `turn-in: SELF` или `ANY` считается некорректной и **молча не
выдаётся**. Адресат обязателен.

⚠️ **Адресат может не заспавниться.** На матч попадает случайное подмножество
точек NPC. Если у типа-адресата одна-две точки, в конкретном матче его может не
оказаться — темка станет невыполнимой, а игрок заблокирован до броска с
кулдауном. **Ставьте адресатам передачек 4–5 точек.**

### `DELIVERY` против `FIND`

Оба проверяют инвентарь в момент сдачи. Разница одна: `DELIVERY` **забирает**
предметы, `FIND` — нет.

### «Волшебный ключик»

Крюк, выпадающий с шансом 10 % за **любую** сданную темку. ПКМ по игровому
сундуку — мгновенный рефилл новым лутом.

- Содержимое сундука при этом **затирается**, даже если он был полон.
- Промах по не-игровому сундуку **не тратит ключ**.
- Ключ — обычный предмет, **выпадает при смерти**.

### Пример

```
/escape createtheme МАЛЯВА
/escape themetype МАЛЯВА COURIER
/escape themedesc МАЛЯВА <gray>Отнеси передачку Кладовщику. Не вскрывать!
/escape themeamount МАЛЯВА 1
/escape themegold МАЛЯВА 25
/escape themereturn МАЛЯВА КЛАДОВЩИК
/escape addtheme БАРЫГА МАЛЯВА
```

## NPC — торговцы и смотрящие

### Роли определяются содержимым

Отдельного флага нет:

| Есть товары | Есть темки | Роль | ПКМ открывает |
|---|---|---|---|
| ✅ | ❌ | Торговец | Магазин |
| ❌ | ✅ | Смотрящий | Список темок |
| ✅ | ✅ | Совмещённый | Меню выбора «Магазин / Темки» |
| ❌ | ❌ | Пустой | Пустой магазин. `check` даёт WARNING |

### Жёсткие лимиты — превышать нельзя

> **Максимум 28 товаров на тип NPC.**
> При 29 и более открытие магазина игроком **вешает главный поток сервера**
> в бесконечном цикле. Валидатор этого не проверяет, редактор трейдов не
> ограничивает. Следите за размером ассортимента сами.

> **Максимум 7 темок на NPC.**
> Восьмая и далее просто не отображаются в меню, без предупреждения. Уже сданные
> темки продолжают занимать слоты — по ходу матча видимый список сжимается.

### Ассортимент

Добавление — только через GUI:

```
/escape addtrade БАРЫГА
```

Слот 16 — предмет, кнопки вокруг слота 12 — цена (±1 / ±10, диапазон **1–255**),
слот 26 — СОХРАНИТЬ, слот 18 — ОТМЕНИТЬ.

Или руками в `traders.yml`:

```yaml
traders:
  БАРЫГА:
    name: '<gold>Барыга'
    trades:
      - {type: BREAD, amount: 3, price: 2}
      - {type: LAPIS_LAZULI, amount: 4, price: 6}
    themes:
      - МАЛЯВА
```

**Детали покупки:**

- Валюта — **физические золотые слитки** в инвентаре, не виртуальный баланс.
- Нужен **полностью свободный слот** — место в существующем стаке не считается.
- Цена `0` автоматически становится `1`. Бесплатных товаров не бывает.
- Купленный предмет проходит через **износ арены** (см. выше).
- **Удаления товара в GUI нет** — только добавление. Убрать товар можно лишь
  правкой `traders.yml` вручную с последующим `/escape reload`.

**Тихие пропуски:** неверный материал или отсутствующий/нечисловой `price` —
запись выбрасывается при загрузке **без единой строки в логе**. Товар просто не
появится в магазине.

### Совмещённый NPC

```
/escape createvillager БАРЫГА
/escape villagername БАРЫГА <gold>Барыга
/escape addtrade БАРЫГА          ← товары
/escape addtheme БАРЫГА МАЛЯВА   ← темки
```

## Рычаги — навигация и задания

```
/escape addlever ТЮРЬМА Столовая
```

Две роли сразу:

1. **Цель заданий типа `ACTIVATE`** — поле `idle` контракта или темки должно
   **посимвольно совпадать** с именем рычага.
2. **Пункт меню «Места»** Личного проводника — игроки ориентируются по карте.

Ставьте рычаги в узнаваемых местах и называйте их так, как игроки называют эти
места вслух.

## Руды

```
/escape addore ТЮРЬМА
```

- ⚠️ **Используются ВСЕ точки, каждый матч.** Ограничивающего параметра нет.
- Материал выбирается случайно при старте: уголь ~26 %, железо ~21 %,
  заражённый камень ~21 %, золото ~16 %, алмаз ~11 %, изумруд ~5 %.
- **Руда не даёт ни предметов, ни опыта.** Она нужна для контрактов и темок
  `MINE`/`BREAK`, статистики «руд добыто» и события «Подкоп».
- Заражённый камень выпускает чешуйницу — это ванильное поведение и часть
  задумки.

## Проверка контента

```
/escape check ТЮРЬМА
```

Валидатор ловит битый контент: несуществующие материалы, отсутствующие рычаги,
`COURIER` без адресата, темки и контракты без типа, NPC без содержимого.

⚠️ **Валидатор проверяет темки только у тех типов NPC, которые фактически
расставлены на этой арене.** Темки у неиспользуемых типов остаются
непроверенными — их сломанность вскроется, только когда вы поставите такой NPC.

Полный список правил — [01-commands.md](01-commands.md#админ-проверка-арены).

## Рецепт стартового контент-пака

Минимум, который делает матч осмысленным:

| Элемент | Количество | Зачем |
|---|---|---|
| Позиций в пуле лута | 30–50 | Разнообразие находок |
| Контрактов на арене | 5–8 разных типов | Занятие помимо драки |
| Типов NPC | 3–5 | Экономика и цели для перемещения |
| Темок | 8–15 | Живой мир, риск с передачками |
| Именованных рычагов | 5–10 | Навигация + `ACTIVATE`-задания |
| Рудных точек | 20–60 | `MINE`-задания и событие «Подкоп» |

Готовые примеры — [`traders-prison.yml`](../examples/traders-prison.yml),
[`themes-prison.yml`](../examples/themes-prison.yml),
[`loot-prison.yml`](../examples/loot-prison.yml).
