# Настройка и балансировка арены

Тайминги, экономика, события, блоки возрождения, оффлайн-стражи. Что на что
влияет, где скрытые минимумы и готовые пресеты.

Справочник ключей — [02-configs.md](02-configs.md).

---

## Как на самом деле считаются таймеры

**Это главное, что нужно понять перед настройкой.**

Матч тикает раз в секунду, счётчик `remaining` идёт **вниз** от
`duration-seconds` до нуля. Периодические события срабатывают, когда **остаток**
делится на интервал нацело — а не по прошедшему времени.

Отсюда формула количества срабатываний за матч:

```
N = floor((duration − 1) / interval)
```

### Пример на дефолтах

| Параметр | Значение | Срабатываний | Когда (остаток) |
|---|---|---|---|
| `duration` | 1200 | — | — |
| `salaryinterval` | 600 | **1** | 600 |
| `eventinterval` | 210 | **5** | 1050, 840, 630, 420, 210 |
| `lightning-interval` | 300 | **3** | 900, 600, 300 |

> ⚠️ **На заводских настройках зарплата выдаётся ровно один раз за матч.**
> Если вы ожидали «каждые 10 минут дважды за 20-минутный матч» — этого не
> происходит. Поставьте `salaryinterval 300`, чтобы получить три выплаты.

### Скрытые минимумы

Значения ниже этих порогов **молча поднимаются** — в конфиге останется ваше
число, а работать будет минимум:

| Параметр | Жёсткий минимум |
|---|---|
| `salaryinterval` | 60 сек |
| `eventinterval` | 30 сек |
| `respawn-block.lightning-interval-seconds` | 60 сек |
| `respawn-block.respawn-delay-seconds` | 1 сек |
| `offline-guard.zombie-seconds` | 10 сек |
| `offline-guard.return-seconds` | `zombie-seconds + 10` |

### Свечение — не интервал

`glowtime` — это **точное совпадение** остатка, срабатывает ровно один раз.
Если `glowtime >= duration`, свечения не будет никогда.

> ⚠️ **Заводские настройки дают WARNING валидатора.** При `duration 1200` и
> `glowtime 600` условие «`glowtime` ≥ половины длительности» выполняется, и
> `/escape check` предупреждает на каждой свежесозданной арене. Либо уменьшите
> `glowtime`, либо примите это осознанно.

### Финальная битва бесконечна

На нуле таймера главный тик **останавливается навсегда**. После этого:

- не идут зарплаты, свечение, события и молнии по блокам возрождения;
- **у финальной битвы нет таймаута** — если игроки не убивают друг друга, матч
  висит, пока админ не сделает `/escape stop`.

Учитывайте это, если ставите большие `duration` на публичном сервере.

## Тайминги

| Параметр | Дефолт | На что влияет |
|---|---|---|
| `duration` | 1200 | Время до финальной битвы. Задаёт масштаб всему остальному |
| `startdelay` | 60 | Отсчёт при наборе `min-players` |
| `startdelayfull` | 10 | **Урезает уже идущий отсчёт**, когда лобби заполнилось |
| `eventinterval` | 210 | Плотность случайных событий |
| `salaryinterval` | 600 | Периодичность дохода |
| `glowtime` | 600 | Когда все становятся видимыми |

### Тонкость `startdelayfull`

Он **не запускает** отсчёт, а сокращает текущий — и только в момент **входа
игрока**, доведшего лобби до максимума. Следствия:

- Если `startdelayfull > startdelay`, механика не сработает никогда.
- Если лобби заполнилось не через вход (например, изменили `max-players`),
  сокращения не будет.

## Экономика

### Источники золота

| Источник | Количество | Частота |
|---|---|---|
| Старт (`startgold`) | 24 | Один раз |
| Зарплата (`salarygold`) | 16 | По формуле выше |
| Бонус свечения (`glowgold`) | 18 | Один раз |
| Контракт | `price` контракта | За выполнение |
| Темка | `gold` темки | За сдачу |
| Слом чужого блока возрождения | 8 / 12 / 16 / 24 / 32 | По уровню блока |
| Событие «Снять броню» | +8 | Случайно |
| Событие «Подкоп» | +6 | Случайно |

### Куда золото тратится

| Трата | Стоимость |
|---|---|
| Покупки у торговцев | Цена товара (1–255) |
| Прокачка блока: медь → железо | 32 |
| железо → золото | 56 |
| золото → алмаз | 128 |
| алмаз → изумруд | **3 убийства**, золото не тратится |
| Событие «Взятка» | −5 (даёт случайный предмет из лута) |

**Полная прокачка блока возрождения стоит 216 золота** плюс 3 убийства. Это
осознанный «шпиль» экономики: сверяйте с ним доход.

### Прикидка дохода за матч (дефолты)

```
24 (старт) + 16×1 (зарплата) + 18 (свечение) = 58 золота пассивно
```

То есть без контрактов, темок и убийств игрок не доберётся даже до железного
уровня (32) с запасом на покупки. **Основной доход должен идти от заданий** —
если контрактов и темок мало, экономика встанет.

Ориентир: чтобы игрок мог дойти до алмаза, суммарная награда доступных ему
заданий должна составлять 150–250 золота за матч.

## Лут и износ

| Параметр | Дефолт | Эффект |
|---|---|---|
| `chests` | 75 | Сколько сундуков активируется на матч |
| `wearmin` / `wearmax` | 40 / 90 | Диапазон случайного износа, % |
| `dynamicchests` | 0 | Немаркированные сундуки карты становятся игровыми |
| `chest-refill-seconds` (глобально) | 180 | Через сколько пустой сундук пополняется |

**Рефилл срабатывает только для полностью пустого сундука.** Разобранный
наполовину не пополнится, пока игроки не выгребут остаток.

Настройка сложности через износ:

| Стиль | `wearmin` / `wearmax` |
|---|---|
| Хардкор, вещи рассыпаются | 70 / 95 |
| Классика | 40 / 90 |
| Мягко | 10 / 50 |
| Износ выключен | `wearmax 0` |

## Снежки, яйца и удочка-крюк

Глобальные, не per-arena (`config.yml`). Подробности — [02-configs.md](02-configs.md#снежки-яйца-и-удочка-крюк).

| Параметр | Дефолт | Эффект |
|---|---|---|
| `projectiles.snowball-damage` | 2.0 | Урон снежка, 2.0 = одно сердце |
| `projectiles.egg-damage` | 3.0 | Урон яйца, 1.5 сердца |
| `fishing-hook.base-strength` | 0.55 | Рывок удочки без зачарования |
| `fishing-hook.power-strength-per-level` | 0.30 | Прибавка за уровень «Силы» |
| `fishing-hook.max-strength` | 2.5 | Потолок силы рывка |

**Почему метательное не имба.** Ванильное окно неуязвимости (1 сек) остаётся:
сколько бы снежков ни летело, засчитывается один за секунду, и броня режет урон.
Чтобы кого-то добить, нужен запас и меткость — на дефолтах это ~10 попаданий
снежками или ~7 яйцами по безбронному. Регулировать проще всего весом позиций
в `loot.yml` (сколько снежков вообще есть на карте), а не уроном.

| Стиль | `snowball` / `egg` |
|---|---|
| Метательное решает | 3.0 / 4.0 |
| Классика | 2.0 / 3.0 |
| Только докидывание | 1.0 / 1.5 |
| Ванильное поведение (0 урона) | 0 / 0 |

**Крюк.** Без зачарования — рывок на пару блоков (сбить с бега, стянуть с края).
«Сила» на удочке ванильно не накладывается, поэтому зачарованная удочка — это
редкий предмет, который вы сами кладёте в лут или в магазин; чем выше уровень,
тем дальше летит жертва. Уровень 5 (0.55 + 1.5 = 2.05) тащит через полкарты и
часто добивает падением — если это слишком, снижайте
`power-strength-per-level`, а не `max-strength`.

## События

12 событий, выбираются **равномерно случайно** каждые `eventinterval` секунд.
Повторы не ограничены, истории нет.

| Событие | Окно | Требование | Провал |
|---|---|---|---|
| `ROOF` «Крыша» | 8 с | Встать под крышу | Молния (визуал) + 2 сердца |
| `GRASS` «Трава» | 8 с | Стоять на блоке травы | Слепота 10 с |
| `SNEAK` «Присядь» | 8 с | Сидеть на корточках | 1.5 сердца |
| `GOLD_IN_HAND` | 8 с | **Ровно 10** слитков в руке | Свечение 18 с |
| `NO_ARMOR` | 8 с | Снять всю броню | Ничего (награда +8 золота) |
| `LOCKDOWN` «Отбой» | 8 с | Не сходить с места (±1 блок) | 2 сердца |
| `SEARCH` «Обыск» | 8 с | Выбросить любой предмет | Отбирается случайный предмет |
| `BRIBE` «Взятка» | 8 с | ≥5 слитков в руке | Ничего (обмен 5 золота на предмет) |
| `DIG` «Подкоп» | 20 с | Сломать любую руду | Ничего (награда +6 золота) |
| `BLOOD_MOON` «Кровавая луна» | 120 с | — | Урон PvP ×1.5, всем ночное зрение |
| `FOG` «Туман» | 50 с | Сидеть на корточках | Слепота, пока стоишь |
| `RATION` «Голодный паёк» | мгновенно | — | Сытость режется до 3 полосок |

### Условия и подводные камни

- **`BLOOD_MOON` запускается только ночью** в мире арены. При вечном дне
  событие не выпадет никогда — только через `/escape debug event BLOOD_MOON`.
- Множитель ×1.5 действует **только на PvP между участниками**. Урон от мобов и
  окружения не усиливается.
- **Пока идёт событие, следующая точка интервала проглатывается.** Длинные
  события (`BLOOD_MOON` 120 с, `FOG` 50 с) съедают следующий слот, если
  `eventinterval` меньше их длительности.
- ⚠️ **Событие, не успевшее закрыться до нуля таймера, не завершится никогда.**
  Для `BLOOD_MOON` это значит ×1.5 на всю финальную битву. Держите
  `eventinterval` таким, чтобы последняя точка запуска была дальше 120 секунд
  от конца матча.
- `SEARCH` может отобрать **надетую броню** или предмет из левой руки.
- `GOLD_IN_HAND` требует **ровно 10** слитков — с одиннадцатью игрок провалит
  событие. `BRIBE` требует «не меньше 5». Это частый источник жалоб.
- `/escape debug event <ТИП>` **не печатает анонс** — игроки увидят только
  эффекты. Не путайте с естественным запуском при тестировании.

## Блоки возрождения

Каждый игрок получает на старте **медный блок возрождения с 1 зарядом**.

| Уровень | Цена прокачки | Заряды | Награда сломавшему | Страж при выходе |
|---|---|---|---|---|
| Медь | старт | 1 | 8 | Зомби |
| Железо | 32 золота | +2 | 12 | Кадавр |
| Золото | 56 золота | +2 | 16 | Свинозомби |
| Алмаз | 128 золота | +2 | 24 | Утопленник |
| Изумруд | 3 убийства на алмазе, **6+ живых** | +2 | 32 + Сила II на 30 с | Зомби-житель |

Максимум зарядов за матч — **9**.

### Условия установки

Проверяется **только вертикальная колонка**: два свободных блока сверху,
твёрдая опора снизу. Границы арены, приваты, высота и расстояние до чужих
блоков **не проверяются** — игрок может поставить блок где угодно в мире арены,
включая места под водой.

### Что нельзя настроить

В конфиг **не вынесены**: цены прокачки, награды за слом, число зарядов, порог
«3 убийства», порог «6 живых», длительности эффектов. Меняются только правкой
кода.

### Настраиваемое

| Ключ (config.yml) | Дефолт | Эффект |
|---|---|---|
| `respawn-block.respawn-delay-seconds` | 5 | Сколько игрок ждёт в спектаторах перед возрождением |
| `respawn-block.lightning-interval-seconds` | 300 | Как часто молния подсвечивает все установленные блоки |

### Подводные камни

- **Умер, не установив блок → выбыл окончательно**, предмет уничтожается. Блок
  нельзя выбросить и нельзя спрятать в сундук.
- **Блок выдаётся один раз за матч.** После уничтожения восстановить нечем —
  админ-команды выдачи блока не существует.
- Место установки **раскрывается молнией** сразу при установке и далее
  периодически. Отключить нельзя.
- Убийства, набитые **до** прокачки в алмаз, к изумруду **не засчитываются** —
  счётчик обнуляется при переходе на алмазный уровень.
- Порог «6+ живых» считает и игроков под оффлайн-стражем, и ожидающих
  возрождения — реально на арене может бегать меньше шести.
- Последний заряд: игрок возрождается на блоке, и тут же **молния бьёт по нему
  же** — блок исчезает.
- ⚠️ **Всё состояние блоков живёт в памяти.** Рестарт сервера посреди матча =
  блоки потеряны, а физические блоки останутся стоять на карте.

### «Молния прозрения»

Громоотвод, раскладываемый по сундукам **на старте матча**. ПКМ — молния по
всем установленным блокам сразу.

- Квота: **1 + (число игроков / 2)**. При 12 игроках — 7 штук.
- Раскладывается только по **статическим** точкам сундуков. Арена на одних
  `dynamicchests` не получит ни одного предмета.
- Полные сундуки пропускаются без компенсации — реально может лечь меньше квоты.

## Оффлайн-стражи

Живой игрок вышел из идущего матча — вместо мгновенного выбывания на его месте
остаётся AFK-страж.

| Ключ (config.yml) | Дефолт | Эффект |
|---|---|---|
| `offline-guard.zombie-seconds` | 90 | Сколько страж стоит на карте |
| `offline-guard.return-seconds` | 240 | Общее окно возврата **от момента выхода** |

**Сценарии возврата:**

| Когда вернулся | Что получает |
|---|---|
| Страж ещё жив | Инвентарь **цел**, продолжает игру |
| Страж истёк, есть блок с зарядом | Инвентарь **очищен**, появляется на блоке, тратится заряд |
| Страж истёк, блока нет | Заочное выбывание |
| Стража **убили** | Это смерть игрока: кредит убийце, вещи выпали |

**Условия появления стража:** игрок был жив, матч в фазе `RUNNING`, финальная
битва не началась, игрок не ждал возрождения. Иначе — мгновенное выбывание.

На старте финальной битвы **все стражи ликвидируются**, их владельцы выбывают
заочно.

## Готовые пресеты

### Быстрый матч — 10 минут

Для ротации на публичном сервере.

```
/escape set ТЮРЬМА duration 600
/escape set ТЮРЬМА eventinterval 120      → 4 события
/escape set ТЮРЬМА salaryinterval 150     → 3 выплаты
/escape set ТЮРЬМА salarygold 14
/escape set ТЮРЬМА glowtime 240
/escape set ТЮРЬМА startgold 32
/escape set ТЮРЬМА startdelay 40
/escape set ТЮРЬМА chests 50
```

Экономика ужата — поднимите награды контрактов, иначе до алмаза никто не дойдёт.

### Классика — 20 минут (с исправленной зарплатой)

```
/escape set ТЮРЬМА duration 1200
/escape set ТЮРЬМА eventinterval 210      → 5 событий
/escape set ТЮРЬМА salaryinterval 300     → 3 выплаты
/escape set ТЮРЬМА salarygold 16
/escape set ТЮРЬМА glowtime 420           → без WARNING валидатора
/escape set ТЮРЬМА startgold 24
```

### Долгая осада — 35 минут

```
/escape set ТЮРЬМА duration 2100
/escape set ТЮРЬМА eventinterval 240      → 8 событий
/escape set ТЮРЬМА salaryinterval 420     → 4 выплаты
/escape set ТЮРЬМА salarygold 18
/escape set ТЮРЬМА glowtime 600
/escape set ТЮРЬМА chests 100
/escape set ТЮРЬМА wearmin 50
/escape set ТЮРЬМА wearmax 95
```

Длинный матч = больше дохода = блоки возрождения качаются до изумруда.
Убедитесь, что в пуле лута есть железные и алмазные кирки, иначе изумрудные
блоки будет нечем ломать.

## Чек-лист после изменения настроек

- [ ] `/escape check <ID>` — не появилось новых WARNING
- [ ] Пересчитано число срабатываний по формуле `floor((duration − 1) / interval)`
- [ ] `glowtime` меньше половины `duration`
- [ ] Последняя точка запуска событий дальше 120 сек от конца матча
- [ ] Доход за матч сопоставим со стоимостью прокачки блока (216 золота)
- [ ] В пуле лута есть кирки уровня железа и выше
- [ ] Проведён соло-прогон — [06-operations.md](06-operations.md#соло-тест-арены)
