# Механики

## Золото — валюта

Единственная валюта — **золотые слитки** прямо в инвентаре (никакого скорборд-баланса). Проверка «сколько у игрока золота» — пересчёт слитков по инвентарю; оплата — изъятие слитков.

**Источники золота:**

| Источник | Сколько |
|---|---|
| Стартовый набор | 24 |
| «Зарплата» каждые 10 минут (с молнией-эффектом) | 16 |
| Отметка «10 минут до конца» | 18 |
| Случайное событие «Снимите всю броню» | 8 |
| Выполненный контракт | `price` контракта |
| Лут из сундуков | зависит от пула арены |
| Убийство игрока | его выпавшее золото |

**Трата:** покупки у торговцев.

Важное следствие: добыча руд **не даёт ресурсов** (дроп отключён), т.е. золото нельзя «намайнить» — только лут, контракты, события и отбор у других.

## Сундуки и лут

- На старте матча из всех точек сундуков случайно активируется `chestCount` штук (по умолчанию 75 из настроенных).
- **Генерация лута:** 2–4 предмета в случайные слоты (0–26). Каждый предмет: с шансом **1/4** — бумага-контракт (если у арены есть контракты), иначе — случайный предмет из **взвешенного пула** арены.
- Пул лута: админ добавляет предмет с «весом» 1–250; вес = сколько копий предмета кладётся в пул (чем больше, тем чаще выпадает).
- Под каждым сундуком стоит **невидимая стойка для брони** — служебная сущность:
  - ПКМ по стойке открывает сундук над ней (можно лутать «через блок»).
  - Имя стойки — индикатор состояния сундука.

### Автопополнение (рефилл)

Когда игрок закрывает **полностью пустой** сундук (проверяется сундук, на который игрок смотрит, в радиусе 10 блоков):

1. Стойка показывает табличку **«Пустой»**.
2. Через **3 минуты** сундук наполняется заново, табличка меняется на **«Refilled»**.
3. Повторный запуск таймера на тот же сундук блокируется, все таймеры отменяются при конце матча.

## Контракты (задания)

Контракты — бумаги **«Задание»**, выпадающие из сундуков. Это основной PvE-контент.

**Строение бумаги (все данные хранятся в лоре предмета):**

| Строка лора | Содержание |
|---|---|
| 0 | Скрытый ID контракта (обфусцировано «магическим» шрифтом `§8§k`) |
| 2 | Описание задания |
| 3 | `Прогресс: N` — живой счётчик |
| 4 | `Цена: N` — награда золотом |
| 6 | Скрытые метаданные: `арена/idle/слот` (тоже `§8§k`) |

Прогресс живёт **в самой бумаге**: потерял/выбросил бумагу — потерял прогресс; бумага выпадает при смерти, контракт можно «отобрать».

**Типы контрактов** (задаются админом; `idle` — параметр-цель):

| Тип | Условие прогресса | `idle` |
|---|---|---|
| `KILLS` | Убийства игроков | — |
| `ACTIVATE` | ПКМ по именованному рычагу на арене | имя рычага |
| `MINE` | Сломать блок (руду) | имя материала |
| `BREAK` | Сломать блок | имя материала |
| `FIND` | Иметь предмет при закрытии сундука (+1 за каждый подходящий предмет в инвентаре за каждое закрытие) | имя материала |
| `LOOT` | «Облутать сундуки» — **объявлен, но не реализован** | — |

**Выполнение:** при каждом шаге прогресса в чат идёт «Контракт: X — Прогресс: N/M». По достижении `amount`:

```
=====================================
 Контракт выполнен. Золото получено.
=====================================
```

Бумага исчезает, выдаётся золото (`price`). Счётчик статистики `QuestsCompleted` +1 (для MINE/BREAK/FIND). У FIND-контракта при завершении дополнительно изымается **весь стак** предмета-цели.

## Торговцы

- Жители-NPC на арене (без AI, неуязвимые, с цветным именем типа над головой). На старте матча спавнится `villagersCount` штук (по умолчанию 32) на случайном поднаборе настроенных точек.
- **Типы торговцев** создаются глобально (не per-arena): у типа есть ID, отображаемое имя и список товаров `предмет → цена` (цены 1–255 золота).
- ПКМ по торговцу — GUI **«Магазин <имя>»** (рамка из фиолетовых панелей, размер подстраивается под число товаров). На товаре в лоре дописано «Цена: N золота».
- Покупка: клик по товару → проверка золота и свободного места → списание золота → выдача предмета (без строк цены в лоре) → «Предмет куплен. −N золота. У вас осталось M золота» → статистика `TradesCompleted` +1.

## Руды и заражённый камень

- На каждой рудной точке при старте появляется случайный блок:

| Блок | Вероятность |
|---|---|
| Угольная руда | 5/19 (~26%) |
| Железная руда | 4/19 (~21%) |
| Золотая руда | 3/19 (~16%) |
| Алмазная руда | 2/19 (~11%) |
| Изумрудная руда | 1/19 (~5%) |
| **Заражённый камень (чешуйница!)** | 4/19 (~21%) |

- В матче ломать можно **только** руды (`*_ORE`) и заражённые блоки (`INFESTED_*`) — всё остальное запрещено, кроме железных решёток.
- **Дроп с руд отключён** — добыча нужна для контрактов MINE, а заражённый камень — ловушка «на жадину».
- Руды после матча не восстанавливаются автоматически (точки перегенерируются при следующем старте).

## Железные решётки — «побег»

- Железные решётки (`IRON_BARS`) — единственный «строительный» блок, который можно ломать. Технически ломаются чем угодно, но кирка («Вилка») делает это в разы быстрее — она и есть инструмент побега по замыслу.
- Сломанные решётки запоминаются и **восстанавливаются после матча**.
- Дропа нет. Геймплейно это двери между зонами тюрьмы: выломал — открыл проход (себе и всем остальным).

## «Личный проводник» / «Личный помощник»

Бумага из стартового набора. ПКМ → GUI на 9 слотов «Личный помощник» (заполнитель — лианы) с 4 способностями. Все подсказки **бесплатные**, но с индивидуальными кулдаунами (per игрок, per способность):

| Способность | Иконка | Кулдаун | Эффект |
|---|---|---|---|
| **Ближайший сундук** | сундук | 2 мин 30 сек | Координаты ближайшего сундука (может быть пустой) |
| **Случайный игрок** | голова игрока | 4 мин | Координаты случайного другого игрока (мин. 3 игрока в матче) |
| **Ближайший торговец** | золотой слиток | 3 мин | Координаты ближайшего торговца |
| **Координаты мест** | карта | 1 мин 30 сек | Открывает список «отмеченных локаций» арены; клик по локации — её координаты в чат |

- «Отмеченные локации» — это именованные рычаги, расставленные админом (двойное назначение: цели ACTIVATE-контрактов и точки интереса карты — «Столовая», «Прачечная» и т.п.).
- Координаты выдаются в чат в формате `X | Y | Z`.
- При попытке использовать раньше времени: «Вы недавно уже использовали это. До следующего раза: N сек.»

## Случайные события

Каждые **210 секунд** (~3,5 минуты) объявляется случайное событие из 5. Через **8 секунд** — проверка каждого живого игрока.

| # | Объявление | Проверка через 8 сек | Наказание/награда |
|---|---|---|---|
| 0 | «Встаньте **под крышу**, иначе по вам ударит молния» | Колонна блоков над головой (см. баги: проверка фактически инвертирована) | Эффект молнии; «Вы не встали под крышу...» / «Вы избежали молнии!» |
| 1 | «Встаньте на **траву**, иначе получите слепоту» | Блок под ногами — трава (см. баги: в 1.16 проверяется не тот материал) | Слепота (см. баги: длительность в тиках) |
| 2 | «**Присядьте**, иначе потеряете полтора сердца» | Игрок крадётся | 3.0 урона |
| 3 | «Возьмите в руку **ровно 10 золота**, иначе будете светиться» | В основной руке стак из ровно 10 золотых слитков | Свечение (см. баги) |
| 4 | «Снимите **всю броню** — получите 8 золота» | Все 4 слота брони пусты | Награда +8 золота (наказания нет) |

События вносят хаос: заставляют двигаться, выдавать позицию (свечение), рисковать (снять броню в PvP-игре) или платить вниманием.

## PvP и статистика матча

- Урон между игроками в матче обычный (после лобби).
- Убийства пишутся в статистику `Kills` убийцы; контракт KILLS двигается по прогрессу (см. баги — награда уходит не тому).
- Per-match статистика арены: `Kills`, `QuestsCompleted`, `TradesCompleted` — выводится при победе.
- В классе `User` заготовлены глобальные счётчики (wins/loses/kills/deaths/oresMined/questsCompleted/mvpGames/gamesPlayed/lastGameKills/bestGameKills) — **почти нигде не обновляются** (задел на будущее, `/escape stats` так и не реализован).

## Защита арены и ограничения

В матче запрещено:

- Ставить блоки; ломать всё, кроме руд/заражённых/решёток.
- Открывать/использовать: верстак, печку, наковальню, эндер-сундук, шалкеры.
- Ломать рамки и висячие сущности; вытаптывать посевы (PHYSICAL отменяется, кроме нажимных плит).
- Использовать команды (кроме `leave` / `stop` / `start`).

Для миров, где есть арены (постоянно, не только в матче):

- Огонь: не распространяется, блоки не поджигаются (вне матча) и не горят.
- Листва не опадает.

В лобби дополнительно: не тратится голод, нельзя выбрасывать предметы.

## Сохранение и восстановление игрока

Перед входом в лобби в файл `Players/<uuid>.data` пишутся: позиция, displayName, инвентарь, броня, HP, еда, полёт и allowFlight, гейммод, активные эффекты, уровень, опыт, скорость ходьбы/полёта. После матча (или при выходе/перезаходе) всё восстанавливается 1-в-1. Участие в мини-игре полностью «бесследно» для основного выживания.
