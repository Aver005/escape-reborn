# Админка: команды, настройка арены, хранение данных

Команда: `/escape` (алиасы: `es`, в коде 2021 также `ecs` — опечатка от `esc`). Права админа: `escape.admin`. Есть tab-completion (с 2021): подкоманды + подстановка ID арен.

## Команды игрока

| Команда | Действие |
|---|---|
| `/escape` | Открыть GUI-меню арен |
| `/escape help` | Справка (красиво оформленная рамками) |
| `/escape join` | Открыть GUI-меню арен |
| `/escape join <ID>` | Зайти на конкретную арену |
| `/escape leave` | Покинуть матч/лобби |
| `/escape stats [игрок]` | В help заявлена, **не реализована** |
| `/escape info` | В help заявлена, **не реализована** |

## Команды администратора

### Управление плагином и аренами

| Команда | Действие |
|---|---|
| `/escape save` | Сохранить всё на диск |
| `/escape reload` | Перечитать всё с диска |
| `/escape create <ID> [maxPlayers]` | Создать арену (2–128 игроков, по умолчанию 12). Мир арены = текущий мир админа, лобби = его позиция |
| `/escape remove <ID>` | Удалить арену |
| `/escape enable <ID>` / `disable <ID>` | Включить/выключить арену (выключенная не видна в меню) |
| `/escape stop <ID>` | Принудительно остановить матч (отсчёт 5 сек) |
| `/escape start <ID>` | В help заявлена, **не реализована** (старт только автоматический) |
| `/escape list` | В help заявлена, **не реализована** |
| `/escape setlobby <ID>` | Точка лобби = позиция админа (центр блока) |
| `/escape setname <ID> <имя...>` | Отображаемое имя арены (поддержка `&`-цветов) |
| `/escape setdesc <ID> <текст...>` | Описание арены для меню |
| `/escape setmaxplayers <ID> <N>` | Максимум игроков (2–52) |
| `/escape setminplayers <ID> <N>` | Минимум для автостарта (1–52) |
| `/escape savetoyml <ID>` / `loadfromyml <ID>` | Экспорт/импорт арены в человекочитаемый YML (`Arenas/<ID>.yml`) |

После `create` плагин подсказывает следующую команду цепочки настройки (`setlobby` → `setname` → `addspawn` → ...) — встроенный «мастер настройки».

## Настройка арены маркерами

Ключевая фишка редактора: `add*`-команды выдают **предмет-маркер** с именем «Поставьте, чтобы добавить...» и лором (арена, тип). Админ ставит блок в нужном месте — координата записывается в арену, в чат приходит подтверждение с координатами. Сами блоки-маркеры (кроме рычага и камня) физически не ставятся.

| Команда | Маркер-предмет | Что добавляет |
|---|---|---|
| `/escape addspawn <ID>` | Маяк | Точка спавна игроков |
| `/escape addchest <ID>` | Сундук | Точка сундука |
| `/escape addvillager <ID> <VID>` | Верстак | Точка торговца типа `VID` |
| `/escape addlever <ID> <имя>` | Рычаг | Именованный рычаг (цель ACTIVATE-контрактов + «отмеченная локация» для проводника). Блок ставится реально |
| `/escape addore <ID>` | Камень | Рудная точка. Блок ставится реально |
| `/escape addtable <ID>` | Стол зачарований | Точка стола (**фича не доделана** — см. недоделки) |

Количество активируемых на матч точек: `chestCount` (75), `villagersCount` (32), `tablesCount` (5) — поля есть, команд для их изменения нет (только YML).

## Пул лута сундуков

| Команда | Действие |
|---|---|
| `/escape additem <ID> <вес 1–250>` | Добавить предмет из руки в пул лута с весом (вес = число копий в пуле, больше вес → чаще выпадает) |
| `/escape edititems <ID> <шанс 1–100>` | GUI-редактор пула (54 слота): выложить предметы, закрыть — сохранится. Параметр «шанс» фактически не используется |

## Торговцы (глобальные типы)

| Команда | Действие |
|---|---|
| `/escape createvillager <VID>` | Создать тип торговца |
| `/escape villagername <VID> <имя...>` | Отображаемое имя (с `&`-цветами) |
| `/escape addtrade <VID>` | GUI добавления товара: слот 12 — золото-цена (кнопки ±1/±10, диапазон 1–255), слот 16 — продаваемый предмет, кнопки СОХРАНИТЬ/ОТМЕНИТЬ |

## Контракты (глобальные)

Цепочка создания (каждая команда подсказывает следующую):

| Команда | Действие |
|---|---|
| `/escape createcontract <CID>` | Создать контракт |
| `/escape contracttype <CID> <тип>` | `KILLS` / `ACTIVATE` / `MINE` / `FIND` / `BREAK` / `LOOT` |
| `/escape contractidle <CID> <idle>` | Цель: имя материала (MINE/BREAK/FIND) или имя рычага (ACTIVATE) |
| `/escape contractdesc <CID> <текст...>` | Описание (видно на бумаге) |
| `/escape contractamount <CID> <N>` | Сколько раз выполнить |
| `/escape contractprice <CID> <N>` | Награда золотом |
| `/escape addcontract <ID арены> <CID>` | Привязать контракт к арене |

## Хранение данных

- `plugins/Escape/config.yml` — создаётся, но **пустой** (не используется).
- Всё состояние — **Java-сериализация** (BukkitObjectStream) по папкам:
  - `Arenas/<ID>.arena` — арены (вместе со всеми точками, пулом лута, статистикой!);
  - `Traders/<VID>.trader` — типы торговцев;
  - `Contracts/<CID>.contract` — контракты;
  - `Users/<ник>.user` — пользователи;
  - `Players/<uuid>.data` — снапшоты состояния игроков на время матча.
- Сохранение: `onDisable` и `/escape save`. Загрузка: `onEnable` и `/escape reload`.
- Дополнительно: экспорт/импорт арен в YML (`savetoyml` / `loadfromyml`) — задел на миграцию с бинарного формата.

**Важно для порта:** сериализация завязана на классы плагина и версию Bukkit (сериализованные `Location`, `ItemStack` внутри `HashMap<String, Object>`); старые `.arena`-файлы на новой версии не поднимутся — формат хранения придётся менять (YML/JSON/БД).
