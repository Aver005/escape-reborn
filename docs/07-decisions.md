# Утверждённые решения (согласовано 2026-07-18)

Ответы на вопросы из `06-reborn-questions.md`. Это рабочая спецификация новой версии.

## 1. Стек

| Что | Решение |
|---|---|
| Ядро | **Paper 26.1.2** — последняя стабильная на 18.07.2026 (линейка 26.2 пока beta). Артефакт: `io.papermc.paper:paper-api:26.1.2.build.74-stable` |
| Java | **25** — согласована была 21, но Paper 26.x требует минимум Java 25 (см. docs.papermc.io); toolchain качается Gradle'ом автоматически |
| Сборка | **Gradle 9.6.1, Kotlin DSL**, foojay-resolver для JDK |
| Язык | Java |
| Тексты | Adventure + **MiniMessage**, все строки в `messages.yml` (русский по умолчанию) |

## 2. Данные

- Старые данные **не мигрируем** — всё создаётся с нуля.
- **Каждая арена — отдельная папка**: `plugins/Escape/arenas/<id>/` с файлами `arena.yml` (настройки), `locations.yml` (точки), `loot.yml` (пул лута).
- Глобальные: `traders.yml` (типы торговцев), `contracts.yml` (контракты), `config.yml`, `messages.yml`.
- Статистика игроков — **SQLite** (`stats.db`).
- Снапшоты состояния игроков — `snapshots/<uuid>.yml` (восстановление в т.ч. после краша сервера).

## 3. Баги оригинала

**Фиксим все 16** (см. `05`). Уточнения:

- Слепота (событие «трава») — **10 секунд**; свечение (событие «10 золота») — 18 секунд.
- Проверка «крыши» — честный рейкаст колонны до неба; «трава» — `GRASS_BLOCK`.
- `FIND` — «найти нужный предмет в нужном количестве»: контракт выполняется, когда у игрока одновременно ≥ N предметов-целей (прогресс = максимум собранного). Предметы **не изымаются**.
- **Вилка одноразовая** (как в 2020): 1 использование; число использований настраивается per-arena (`fork-uses`, по умолчанию 1).
- Награда KILLS-контракта — **убийце**. Кредит убийства учитывает снаряды и «последнего ударившего» (10 сек).

## 4. Недоделки

| Фича | Решение |
|---|---|
| Контракт `LOOT` | **Делаем**: +1 за каждый впервые открытый игроком сундук |
| Столы зачарований | **Делаем**: спавн `tableCount` столов на матч, зачарование разрешено (уровни фактически бесплатны из-за XP-таймера — валютой служит лазурит, который админ выставляет у торговцев) |
| NPC с задачами | **НЕ делаем** |
| Глобальная статистика + `/escape stats`, `info`, `list`, `start` | **Делаем** (SQLite) |
| Разминка: лидерборд | **Делаем топ-3** |

## 5. Геймплей

1. **Финальная битва**: отдельные точки `addfinalspawn`; если не заданы — обычные спавны.
2. **Длительность матча**: per-arena, по умолчанию **20 минут**.
3. **Сетка событий/зарплат**: per-arena конфиг (`event-interval` 210 c, `salary-interval` 600 c, суммы — тоже в конфиге).
4. **Спектатор**: ванильный Spectator.
5. **Автостарт с задержкой**: набрали минимум → отсчёт **60 сек**; арена заполнилась → отсчёт сокращается до **10 сек**. Настраивается per-arena.
6. **Новые случайные события**: список предложений — ниже, ждёт согласования. 5 старых (исправленных) — в MVP.

## 7. Блоки возрождения (СОГЛАСОВАНО 2026-07-18)

Upgrade-версия механики из утраченной версии 1.x: каждому игроку на старте
выдаётся **блок возрождения**. Пока блок установлен и заряды не кончились —
смерть не выбивает из матча, игрок возрождается на блоке (вещи выпадают как
обычно).

| Уровень | Блок | Как получить | Возрождений | Награда сломавшему |
|---|---|---|---|---|
| 1 | **Медный** | Выдаётся на старте | 1 | +8 золота |
| 2 | **Железный** | Прокачка за 32 золота | +2 | +12 золота |
| 3 | **Золотой** | Прокачка за 56 золота | +2 | +16 золота |
| 4 | **Алмазный** | Прокачка за 128 золота (2 стака) | +2 | +24 золота |
| 5 | **Изумрудный** | ТОЛЬКО 3 убийства после прокачки до алмазного; недоступен при < 6 живых игроках | +2 | +32 золота **и Сила II на 30 сек** |

Правила:

- Прокачка — через **меню по ПКМ** по своему блоку.
- Блок можно **переносить**: сломал свой → предмет в инвентарь (уровень и
  заряды сохраняются) → поставил в другом месте.
- Условия установки: **2 свободных блока сверху + твёрдый блок снизу**
  (нельзя крепить на стены/потолок).
- Заряды кончились → блок исчезает; **у каждого уровня своя анимация
  исчезновения** (частицы + звук).
- **Молния бьёт в блок**: при установке, каждые **5 минут** матча, при
  уничтожении (врагом / по истечении зарядов / при аннуляции) и при
  использовании «Молнии прозрения».
- **Финальная битва**: блоки всех игроков аннулируются (с молнией и анимацией),
  дальше смерть окончательна.
- **«Молния прозрения»** — редкий предмет из сундуков: на матч гарантированно
  раскладывается **1 + 1 за каждую пару игроков** (напр., 8 игроков → 5 шт).
  Использование: молния бьёт по всем установленным блокам возрождения,
  раскрывая их позиции.

**Отложенное возрождение** (правка владельца 2026-07-18): после смерти игрок
**5 секунд** (конфиг) проводит в спектаторах и только потом появляется на блоке
без вещей. Заряд резервируется в момент смерти. Если за эти 5 секунд блок
уничтожили — возрождение срывается, игрок выбывает окончательно. В окне
ожидания игрок «мёртв»: не участвует в случайных событиях и не получает
зарплату.

Детали реализации (утверждаются постфактум): возрождение — телепорт на блок,
полное HP/еда + Сопротивление II на 5 сек (анти-кемпинг); если к моменту
возрождения все уже подсвечены — свечение вешается и на возрождённого; умер без
установленного блока (в т.ч. нося его в инвентаре) — выбыл окончательно, сам
блок-предмет при этом уничтожается; чужой блок-предмет поставить нельзя;
выброс блока из инвентаря вручную заблокирован.

## 8. Оффлайн-страж: выход/краш живого игрока (СОГЛАСОВАНО 2026-07-18)

Игрок вышел/крашнулся/вылетел, будучи **живым** в идущем матче:

- На его месте появляется **AFK-зомби**: не двигается, **не горит на солнце**,
  в полной экипировке игрока (броня + предмет в руке) и с его ником над головой.
- **Вид стража зависит от уровня блока возрождения игрока** (согласовано
  2026-07-18; нет блока — как медный): медный — зомби, железный — кадавр,
  золотой — свинозомби, алмазный — утопленник, изумрудный — зомби-житель.
- Зомби стоит **90 секунд**:
  - игрок перезашёл, пока зомби жив → продолжает игру как ни в чём не бывало
    (зомби исчезает, инвентарь цел);
  - 90 секунд истекли: зомби исчезает **с анимацией**, вещи выпадают на землю.
    Если блока возрождения нет — игрок **выбыл**. Если блок установлен — при
    входе игрока спавнит на блоке, **если он зашёл не позднее чем через
    4 минуты** (окно считается от выхода); иначе выбыл.
- **Зомби убит другим игроком** → это смерть игрока: убийце кредит (счёт,
  контракты, прогресс изумрудного), вещи выпадают, в чат — сообщение о смерти.
  Дальше действуют обычные правила смерти: установленный блок спасает
  (возврат в 4-минутное окно → спавн на блоке), без блока — выбыл.

Детали реализации (постфактум): спавн на блоке при возврате тратит заряд как
обычное возрождение (и блок исчезает, если заряд был последним); выход во время
5-секундного ожидания возрождения, во время финальной битвы или завершения
матча — выбытие сразу, без зомби; при старте финальной битвы активные
оффлайн-стражи ликвидируются (вещи выпадают, игрок выбывает); окна 90 сек /
4 мин настраиваются в config.yml (`offline-guard`).

## 10. NPC 2.0 — «Смотрящие» и темки (СОГЛАСОВАНО 2026-07-18)

Отменяет прежнее решение §4 «новых NPC не делаем». Единый каркас NPC: тип NPC
может быть **торговцем, смотрящим или совмещать обе роли** (у совмещённого при
ПКМ — выбор «Магазин / Темки»).

**Смотрящий** раздаёт **темки** — задания с шагом сдачи:

- Взял темку → она активна; **одна активная темка на игрока**. Бросить можно,
  но новая — только после кулдауна (60 сек), чтобы не рероллить бесплатно.
- Отличие от бумаг-контрактов: контракт награждает мгновенно, темку надо
  **сдать смотрящему** — дойти живым до сдачи и есть риск.
- **Кому сдавать — свойство темки**: `SELF` (тому же жителю, что выдал),
  `ANY` (любому смотрящему), либо конкретный тип NPC (для «Передачки»).
- **Типы темок**: 6 контрактных (KILLS/ACTIVATE/MINE/FIND/BREAK/LOOT) + два
  новых, чисто NPC-шных:
  - **«Доставка»** — принести смотрящему N предметов, при сдаче они
    **изымаются** (в отличие от FIND);
  - **«Передачка»** — отнести пакет конкретному смотрящему; пакет — предмет
    в инвентаре, умер — выронил, кто подобрал — тот и может сдать.
- **Награда**: золото (у каждой темки своё) + **10% шанс** получить
  «Волшебный ключик» (за любую сданную темку).

**«Волшебный ключик»** — предмет-крюк (tripwire hook): ПКМ по игровому сундуку
→ сундук **немедленно перезаполняется** новым лутом (старое содержимое
перезаписывается). Одно использование, после которого предмет пропадает.

Настройка: темки — глобальный реестр (themes.yml, аналогично контрактам),
привязываются к типам NPC; точки NPC на арене — прежними маркерами торговцев.

## 11. Лут и сундуки (СОГЛАСОВАНО 2026-07-18/19)

- **Лут «бомжеватый»**: поломанные инструменты; броня максимум железная,
  очень изношенная; оружие максимум каменное (ради прикола допустимы алмазные
  лопаты/мотыги). Референс-пул: docs/examples/loot-prison.yml.
- **Случайный износ**: параметры арены wearmin/wearmax %% (дефолт 40-90,
  0 = выкл) — каждому предмету с прочностью при генерации раскатывается
  случайный износ в пределах; ручной урон из additem не перекатывается.
- **Динамические сундуки** (флаг арены dynamicchests, дефолт выкл):
  не отмеченный маркером сундук при первом открытии в матче генерирует лут и
  становится игровым (рефилл/LOOT/ключик); исходное содержимое возвращается
  после матча.


## 12. Улучшенные ванильные механики (СОГЛАСОВАНО 2026-07-19)

Внутри матча часть ванильных механик работает сильнее, чем в обычной игре.
Вне сессии поведение остаётся ванильным.

- **Снежки и яйца наносят урон.** Ванильно оба бьют игрока на 0 — в матче
  снежок снимает **1 сердце**, яйцо **1.5 сердца** (яйца сильнее снежков).
  Урон проходит через броню и через обычный конвейер урона: работает множитель
  «Кровавой луны», засчитывается кредит убийце, срабатывает фейковая смерть.
  Ванильное окно неуязвимости (1 сек) сохраняется — спам в упор не складывается,
  убить можно, только если затариться и попадать. Числа —
  `config.yml` → `projectiles`.
- **Вылупление кур из яиц в мирах арен отключено**: яйцо — оружие, а не
  инкубатор, иначе после матча карта остаётся с курами.
- **Удочка тянет игроков («крюк»).** Попал крючком по игроку и подсёк —
  игрока рывком тащит к рыбаку по дуге. Сила рывка зависит от уровня
  зачарования **«Сила» (POWER)** на удочке; без зачарования удочка тоже
  работает, но слабо (рывок на пару блоков). Формула и потолок —
  `config.yml` → `fishing-hook`.
- Если подцепленный разбился насмерть в течение обычного окна кредита —
  убийство засчитывается рыбаку.
- «Силу» на удочку ванильно не наложить (ни стол, ни наковальня). Такие удочки
  выдаёт админ: `enchants: {power: N}` в `loot.yml`/`traders.yml` либо
  `/give <ник> fishing_rod[enchantments={power:N}]`.


## 13. Стартовые наборы — «касты» (СОГЛАСОВАНО 2026-07-20)

Раньше всем на старте выдавалось одинаковое снаряжение. Теперь есть **касты**
(starter kits) — разнообразие стартового набора под стиль игрока.

- **Выдача — выбор в лобби.** Игрок жмёт сундук-селектор рядом с «Выйти» и
  выбирает каст в GUI. В меню всегда есть **«Без касты»** (только базовое) и
  **«Случайная»**. Не выбравший получает `default-kit` арены (по умолчанию
  `random`; настраивается `/escape kit default`).
- **Каст = доп. предметы + своё стартовое золото.** Системные предметы (вилка,
  «Личный проводник», блок возрождения) выдаются ВСЕГДА, независимо от каста.
  Золото каста перекрывает `start-gold` арены (поле `gold`; `-1` = золото арены).
  Броня каста авто-надевается в слоты.
- **Всё снаряжение выдаётся почти сломанным** (поле `damage` в спеке предмета).
  Это осознанный двигатель игры: постоянный **ремонт снаряжения**. Уровень —
  «бомжеватый» (см. §11): максимум камень/железо, сильно изношено. Разница между
  кастами — не в силе, а в **роли** (ПвП ближний/дальний/контроль, добыча,
  экономика). Золото — минимальное (4–12).
- **Глобальная библиотека + копии на арену.** Шаблоны — в `plugins/Escape/kits.yml`
  (read-only, рукописный формат предметов = `loot.yml`). На арену каст ставится
  **копией** (`/escape kit copy <id> <арена>`) в `arenas/<арена>/kits.yml`, где
  редактируется отдельно, не задевая оригинал. Пять стандартных: `warrior`,
  `archer`, `angler`, `miner`, `dealer`.
- **Редактирование:** предметы — GUI (`/escape kit edit`, как редактор лута);
  имя/иконка(из руки)/золото/дефолт — командами `/escape kit name|icon|gold|default`.
- Если у арены нет ни одного каста — поведение прежнее (всем одинаково),
  селектор в лобби не выдаётся.


## 14. Мусорщик — скупка сломанного (СОГЛАСОВАНО 2026-07-20)

NPC, которому можно **ПРОДАТЬ сломанное снаряжение за золото**. Замыкает петлю
экономики: снаряжение ломается → чинишь на наковальне **или** сдаёшь хлам
Мусорщику → золото → покупаешь/чинишь. Раньше продажи игроком не было вообще.

- **Третья роль NPC** (рядом с магазином и смотрящим): `TraderType.isScavenger()`.
  Один villager может совмещать роли; выбор — через `NpcMenu` (до 3 кнопок).
- **Цена растёт с износом**: `цена = max(1, round(база(материал) x доля_износа)) x
  количество`. Годное почти не платит, убитое — полный скрап. Тематично и
  усиливает петлю ремонта: носи вещь до смерти, потом сдай остаток.
- **Принимает только изношенное**: вещи с прочностью, износ ≥ `min-wear-percent`
  (дефолт 25). Годное отклоняется. Материал должен быть в прайсе Мусорщика.
- **Нельзя продать системное**: вилка, «Личный проводник», блок возрождения,
  бумаги-контракты, пакеты-темки, маркеры — guard по PDC. Только луто-снаряжение.
- **UI — click-to-sell** (`ScavengerMenu`): сканирует инвентарь игрока и
  показывает его сломанные вещи с ценой; клик — мгновенная продажа (предмет из
  инвентаря, потерять в меню нельзя), плюс кнопка «Сдать весь хлам». Звук
  наковальни как фидбэк.
- **Конфиг** — в `traders.yml` секция `scrap` у типа NPC: `min-wear-percent` +
  `prices` (материал → база). Команды: `/escape addscrap <TID> <цена>` (материал
  из руки), `removescrap`, `scrapwear <%>`, `scraplist`. Ликвидация NPC на финале
  — как у прочих.
