# Утверждённые решения (согласовано 2026-07-18)

Ответы на вопросы из `06-reborn-questions.md`. Это рабочая спецификация новой версии.

## 1. Стек

| Что | Решение |
|---|---|
| Ядро | **Paper 26.1.2** — последняя стабильная на 18.07.2026 (линейка 26.2 пока beta). Артефакт: `io.papermc.paper:paper-api:26.1.2.build.74-stable` |
| Java | **25** — согласована была 21, но Paper 26.x требует минимум Java 25 (см. docs.papermc.io); toolchain качается Gradle'ом автоматически |
| Сборка | **Gradle 9.6.1, Kotlin DSL**, foojay-resolver для JDK |
| Язык | Java |
| Тексты | Adventure + **MiniMessage**, все строки в `messages.yml` (русский по умолчанию) |

## 2. Данные

- Старые данные **не мигрируем** — всё создаётся с нуля.
- **Каждая арена — отдельная папка**: `plugins/Escape/arenas/<id>/` с файлами `arena.yml` (настройки), `locations.yml` (точки), `loot.yml` (пул лута).
- Глобальные: `traders.yml` (типы торговцев), `contracts.yml` (контракты), `config.yml`, `messages.yml`.
- Статистика игроков — **SQLite** (`stats.db`).
- Снапшоты состояния игроков — `snapshots/<uuid>.yml` (восстановление в т.ч. после краша сервера).

## 3. Баги оригинала

**Фиксим все 16** (см. `05`). Уточнения:

- Слепота (событие «трава») — **10 секунд**; свечение (событие «10 золота») — 18 секунд.
- Проверка «крыши» — честный рейкаст колонны до неба; «трава» — `GRASS_BLOCK`.
- `FIND` — «найти нужный предмет в нужном количестве»: контракт выполняется, когда у игрока одновременно ≥ N предметов-целей (прогресс = максимум собранного). Предметы **не изымаются**.
- **Вилка одноразовая** (как в 2020): 1 использование; число использований настраивается per-arena (`fork-uses`, по умолчанию 1).
- Награда KILLS-контракта — **убийце**. Кредит убийства учитывает снаряды и «последнего ударившего» (10 сек).

## 4. Недоделки

| Фича | Решение |
|---|---|
| Контракт `LOOT` | **Делаем**: +1 за каждый впервые открытый игроком сундук |
| Столы зачарований | **Делаем**: спавн `tableCount` столов на матч, зачарование разрешено (уровни фактически бесплатны из-за XP-таймера — валютой служит лазурит, который админ выставляет у торговцев) |
| NPC с задачами | **НЕ делаем** |
| Глобальная статистика + `/escape stats`, `info`, `list`, `start` | **Делаем** (SQLite) |
| Разминка: лидерборд | **Делаем топ-3** |

## 5. Геймплей

1. **Финальная битва**: отдельные точки `addfinalspawn`; если не заданы — обычные спавны.
2. **Длительность матча**: per-arena, по умолчанию **20 минут**.
3. **Сетка событий/зарплат**: per-arena конфиг (`event-interval` 210 c, `salary-interval` 600 c, суммы — тоже в конфиге).
4. **Спектатор**: ванильный Spectator.
5. **Автостарт с задержкой**: набрали минимум → отсчёт **60 сек**; арена заполнилась → отсчёт сокращается до **10 сек**. Настраивается per-arena.
6. **Новые случайные события**: список предложений — ниже, ждёт согласования. 5 старых (исправленных) — в MVP.

## 6. Предложения новых случайных событий (НА СОГЛАСОВАНИЕ)

Тюремная тематика, у каждого — простая проверяемая механика:

| # | Название | Механика |
|---|---|---|
| N1 | **«Отбой»** | Не двигайтесь 8 секунд (смещение больше 1 блока = провал) — иначе 2 сердца урона |
| N2 | **«Обыск»** | Выбросьте любой предмет за 8 секунд — иначе случайный предмет из инвентаря выпадает сам |
| N3 | **«Взятка надзирателю»** | Возьмите в руку 5 золота: оно изымается, взамен — случайный предмет из пула сундуков (лотерея) |
| N4 | **«Подкоп»** | Сломайте любую руду за 20 секунд — награда 6 золота (провал не наказывается) |
| N5 | **«Кровавая луна»** | Без условия: 60 секунд весь урон между игроками ×1.5 |
| N6 | **«Туман»** | Без условия: всем слепота на 5 секунд |
| N7 | **«Амнистия»** | Случайный выбывший игрок возрождается на случайном спавне со стартовым набором (если есть выбывшие) |
| N8 | **«Голодный паёк»** | Еда всех игроков опускается до 3 полосок |

Формат ответа: «N1, N3, N5 — берём, N7 — не надо» и т.п.

## 7. Прочие решения реализации (мои, на утверждение постфактум)

- Пакет: `me.aver005.escape`, имя плагина `Escape`, `api-version: 1.21`.
- Прогресс контрактов — в **PersistentDataContainer** бумаги (лор — только отображение). Свойство «бумагу можно отобрать» сохраняется.
- GUI-меню на кастомных `InventoryHolder` (никакого сравнения заголовков).
- Маркеры настройки арен — предметы с PDC-метками (не по имени предмета).
- Таймер: XP-уровень = секунды (как в оригинале, это фишка).
- Смерть — «фейковая» как в оригинале (отмена летального урона, дроп, Spectator), без экрана смерти.
- SQLite-драйвер — встроенный в Paper (`org.sqlite.JDBC`), без shade.
