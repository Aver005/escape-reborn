# Вопросы на согласование перед разработкой

Документы 01–05 описывают «как было». Здесь — развилки, которые нужно решить до начала кода новой версии. По каждому пункту дана рекомендация; жду твоих решений/правок.

## 1. Платформа и стек

| Вопрос | Варианты | Рекомендация |
|---|---|---|
| Ядро | Paper / Spigot / Folia | **Paper 1.21.x** (API богаче, Adventure из коробки; Folia — только если нужны тысячи игроков) |
| Java | 17 / 21 | **21** (требование современных ядер) |
| Сборка | Maven / Gradle | **Gradle (Kotlin DSL)** + paperweight-userdev не нужен (NMS не используется) |
| Язык | Java / Kotlin | **Java** (ближе к оригиналу; Kotlin — если тебе он приятнее) |
| Тексты | ChatColor-строки / Adventure MiniMessage | **Adventure + MiniMessage**, все строки в `messages.yml` (сейчас всё захардкожено по-русски в коде) |

## 2. Хранение данных

Оригинал: Java-сериализация `.arena/.trader/.user/.contract` (хрупко, непереносимо).

- **Рекомендация:** конфиги арен/торговцев/контрактов — **YAML** (человекочитаемо, правится руками); статистика игроков — **SQLite** (готовность к топам/лидербордам).
- Вопрос: нужна ли миграция старых `.arena`-файлов, или арены будут настраиваться заново? (Старых данных, вероятно, нет — уточни.)

## 5. Геймплейные вопросы (нового контента не добавляю без согласования)

1. **Финальная битва:** сейчас это телепорт на обычные спавны без ограничений. Оставить как было, или сделать отдельную зону/сжатие границы? (Минимальный вариант: отдельные точки `addfinalspawn`.)
2. **Длительность матча:** 30 минут — длинно по современным меркам. Оставить 30 / сделать настраиваемой per-arena (рекомендую настраиваемую, по умолчанию 20)?
3. **Сетка событий:** каждые 210 сек, «зарплата» каждые 600 сек — оставить или вынести в конфиг? (Рекомендую конфиг с этими значениями по умолчанию.)
4. **Спектаторы:** оставить ванильный Spectator или сделать «лобби наблюдателя» с телепортами к игрокам?
5. **Автостарт:** сейчас старт мгновенно при достижении минимума. Обычно делают «минимум набран → 60 сек, полная арена → 10 сек». Менять?
6. **Новые случайные события:** 5 старых оставляем; добавлять ли новые (у меня есть идеи) — или сначала 1-в-1?
